<p><b><font face="Courier" size="12">-- ABOUT NAVIGATOR --</font></b></p>
<p><b><font face="Courier" size="12">---------------------</font></b></p>
<p></p>
<p><font face="Courier" size="12">It started as a simple little project. I wanted a way to store a script-in-progress that has compile errors, so I could get the script window out of the way. (This was in 1.1.1) So I built a little palette to store a list of the scripts. Then I got frustrated with editing the scripts of the buttons in a menubar on a Mac, where the buttons don't show. So I added a feature to show a list of the controls on the current card.</font></p>
<p></p>
<p><font face="Courier" size="12">Then the madness set in...</font></p>
<p></p>
<p></p>
<p><b><font face="Courier" size="12">-- NAVIGATOR OVERVIEW --</font></b></p>
<p><b><font face="Courier" size="12">------------------------</font></b></p>
<p></p>
<p><font face="Courier" size="12"> -- Navigator displays lists of Revolution objects: stacks, cards, groups, and controls. </font></p>
<p></p>
<p><font face="Courier" size="12"> -- The list can be of the controls on the current card of the topStack, or of any card or group of any stack. It can also be a list of cards or backgrounds in a stack.</font></p>
<p></p>
<p><font face="Courier" size="12"> -- NEW IN 2.5: The list can now display all the stacks and substacks.</font></p>
<p></p>
<p><font face="Courier" size="12"> -- NEW IN 2.5: The list now supports drag and drop to relayer controls, move them from one place to another, place backgrounds, and more.</font></p>
<p></p>
<p><font face="Courier" size="12"> -- Navigator allows you to edit scripts, set properties or run code against any list of objects.</font></p>
<p></p>
<p><font face="Courier" size="12"> -- NEW IN 2.5: Edit custom properties as well.</font></p>
<p></p>
<p><font face="Courier" size="12"> -- You can also save references to objects in any combination.</font></p>
<p></p>
<p><font face="Courier" size="12"> -- NEW IN 2.5: Rename bookmarks.</font></p>
<p></p>
<p><font face="Courier" size="12" color="#000000"> -- Navigator makes it easy to work with controls that aren't normally accessible: controls in groups, hidden controls, controls on hidden stacks, controls that are in different places.</font></p>
<p></p>
<p><font face="Courier" size="12"> -- NEW IN 2.5: Select a stack to work with by pointing at it.</font></p>
<p></p>
<p><font face="Courier" size="12" color="#000000"> -- Navigator allows you to do things you wouldn't ordinarily be able to do: align controls that are on different cards, for example.</font></p>
<p></p>
<p><font face="Courier" size="12" color="#000000"> -- Navigator makes you a faster developer: edit the scripts of controls on different cards quickly and easily; change any property with a single menu selection rather than using a palette; go to the handler you want to edit in a long script with a single command.</font></p>
<p></p>

<p><b><font face="Courier" size="12">-- CONTROL LIST --</font></b></p>
<p><b><font face="Courier" size="12">------------------</font></b></p>
<p></p>
<p><font face="Courier" size="12">Controls are displayed in a simple list. Bookmarked controls are shown at the top of the list, indented. Below the bookmarked controls is the stack reference in bold. Below the stack reference is either: a single card or background, also in bold, and a list of all the controls on the card/background; or a list of cards or backgrounds in the stack. Alternatively, below the bookmarks is a list of stacks and substacks. If displaying a list of controls, the controls are indented to reflect enclosure within groups on the card. The indent is configurable on the Actions &gt; Prefs menu.</font></p>
<p></p>
<p><b><font face="Courier" size="12">Click</font></b><font face="Courier" size="12"> an object to select it.</font></p>
<p></p>
<p><b><font face="Courier" size="12">Shift-click</font></b><font face="Courier" size="12"> another object to select a range from the one you clicked to the one you shift-clicked. Note there is a slight issue with shift-clicking that manifests if you drag-select upwards through the list, and then try to extend the selection by shift-clicking. The selection can change improperly as a result.</font></p>
<p></p>
<p><b><font face="Courier" size="12">Click and drag</font></b><font face="Courier" size="12"> to select a range of controls.</font></p>
<p></p>
<p><b><font face="Courier" size="12">Command-click</font></b><font face="Courier" size="12"> to toggle the selection of an object.</font></p>
<p></p>
<p><b><font face="Courier" size="12">Control-click</font></b><font face="Courier" size="12"> to pop a contextual menu with various commands. See Contextual Menu (Default) below for details.</font></p>
<p></p>
<p><b><font face="Courier" size="12">Control-Option-click</font></b><font face="Courier" size="12"> to pop a contextual menu of standard properties. You can set them from this menu -- see Contextual Menu (Properties) below.</font></p>
<p></p>
<p><b><font face="Courier" size="12">Control-Option-Command-click</font></b><font face="Courier" size="12"> to pop a contextual menu of custom properties. You can set them from this menu -- see Contextual Menu (Custom Properties) below.</font></p>
<p></p>
<p><b><font face="Courier" size="12">Control-Command-click</font></b><font face="Courier" size="12"> to pop a contextual menu of the handlers in the object's script. You can only do this with a single control hilited. Select any handler in the menu to edit the object's script and go directly to that handler. Option-select any handler to call that handler in the object's script. See Contextual Menu (Handlers) below for details.</font></p>
<p></p>

<p><b><font face="Courier" size="12">-- CONTROL LIST DRAG AND DROP --</font></b></p>
<p><b><font face="Courier" size="12">--------------------------------</font></b></p>
<p></p>
<p><font face="Courier" size="12">This is one of the major new features in Navigator: the list now supports drag and drop for a wide variety of commands. All of them involve the simply action of selecting one or more items in the list, clicking on one of the selected items, and dragging to somewhere else. The context, however, allows for many different results. </font></p>
<p></p>
<p><font face="Courier" size="12">Listed below are the various drag and drop contexts, and the results. Generally, you can mix and match the contexts and Navigator will do the right thing for each item you drag. An example of this is that if you select bookmarks and controls, and drag to another place on the card, Navigator will copy the bookmarks to the current card and rearrange their layers with the controls you dragged, as appropriate.</font></p>
<p></p>
<p><b><font face="Courier" size="12">Anytime</font></b></p>
<p></p>
<p><font face="Courier" size="12">-- Drag bookmarks in the bookmark area to rearrange the bookmark list. This has no effect on the bookmarked objects.</font></p>
<p></p>
<p><font face="Courier" size="12">-- Drag references to objects into the list from elsewhere to create bookmarks to the objects. There is no error checking on this.</font></p>
<p></p>
<p><b><font face="Courier" size="12">When Displaying the Controls on a Card or Group</font></b></p>
<p></p>
<p><font face="Courier" size="12">-- Drag controls in the control list to relayer them.</font></p>
<p></p>
<p><font face="Courier" size="12">-- Drag controls in the control list into a group to move the controls into the group.</font></p>
<p></p>
<p><font face="Courier" size="12">-- Drag controls in a group out of the group to remove the controls from the group.</font></p>
<p></p>
<p><font face="Courier" size="12">-- Drag controls in the control list into the bookmark area of the list to create bookmarks for the selected controls. If bookmarks already exist for a control, no duplicate will be created. This depends on the name of the bookmark, not the underlying ID, so if you have renamed bookmarks, duplicates can be created.</font></p>
<p></p>
<p><font face="Courier" size="12">-- Drag bookmarks of controls (not groups, cards, or stacks; see below for details on them) into the control list to copy the controls to the current card/group. Note that if the bookmarks represent controls on the current card, they will still be copied to the card, duplicating the controls.</font></p>
<p></p>
<p><font face="Courier" size="12">-- Drag bookmarks of cards into the control list to copy the cards to the current stack (no effect if the card is already in the current stack).</font></p>
<p></p>
<p><font face="Courier" size="12">-- If a bookmark is of a group in another stack, drag the bookmarks of groups into the control list to copy the group to the current card. </font></p>
<p></p>
<p><font face="Courier" size="12">-- If a bookmark is of a group (background) in the current stack, drag the bookmarks of the groups into the control list to place the existing backgrounds onto the current card.</font></p>
<p></p>
<p><font face="Courier" size="12">-- Drag bookmarks of stacks into the control list to set the mainstack of the bookmarked stacks to the current stack. This makes the dragged stacks substacks of the current stack. Note that you can easily undo this by switching the display to a substack and dragging the bookmark into the display again. This will make the substack its own mainstack (making it a mainstack again).</font></p>
<p></p>
<p><b><font face="Courier" size="12">When Displaying the Cards in a Stack</font></b></p>
<p></p>
<p><font face="Courier" size="12">-- Drag the cards in the list to set their layer in the stack. This changes the order of the cards in the stack.</font></p>
<p></p>
<p><font face="Courier" size="12">-- Drag cards into the bookmark area to create bookmarks to them.</font></p>
<p></p>
<p><font face="Courier" size="12">-- This one's a bit tricky and perhaps counterintuitive: select one or more cards, and a single bookmark to a group. Click and drag on the bookmark to the group. Drag it anywhere in the card list. This will place the group onto all of the cards you selected in a single command. If the bookmarked group was in another stack, it will first be copied to the current stack, and then placed on all the cards.</font></p>
<p></p>
<p><b><font face="Courier" size="12">When Displaying the Groups (Backgrounds) in a Stack, or the Stack List</font></b></p>
<p></p>
<p><font face="Courier" size="12">-- Drag groups or stacks into the bookmark area to create bookmarks to them.</font></p>
<p></p>

<p><b><font face="Courier" size="12">-- FILTER BOX --</font></b></p>
<p><b><font face="Courier" size="12">------------------</font></b></p>
<p></p>
<p><font face="Courier" size="12">Text typed in the Filter box at the upper right of Navigator acts as a filter on the list. Type "button" to see only buttons, "field" to see only fields, etc. Any text will work, so you can see only objects named "xyz" if you want. The filter acts on whatever is displayed, so you can filter on object names if names are being displayed, or on the ID if IDs are being displayed. Indentation based on groups goes away if the group objects themselves are filtered.</font></p>
<p></p>
<p><font face="Courier" size="12">The filter has no effect when displaying the stack list.</font></p>
<p></p>

<p><b><font face="Courier" size="12">-- ABOUT/SPLASH SCREEN --</font></b></p>
<p><b><font face="Courier" size="12">-------------------------</font></b></p>
<p></p>
<p><font face="Courier" size="12">When Navigator first opens, it displays the About... screen. If you have checked the "I Paid" checkbox, the About... screen will go away automatically as quickly as it appeared. If you haven't checked the "I Paid" checkbox, Navigator waits for you to dismiss the About...screen.</font></p>
<p></p>
<p><font face="Courier" size="12">If you don't see the About... screen when Navigator starts up, try closing Navigator and re-opening it. Navigator shows the About... screen as the last step in its initialization process, so if you don't see it, something went wrong.</font></p>
<p></p>
<p><font face="Courier" size="12">You can dismiss the About... screen by clicking the close button at the upper left, or clicking anywhere in the About... screen.</font></p>

<p></p>
<p><b><font face="Courier" size="12">Prefs</font></b><font face="Courier" size="12"> -- Submenu with options for Navigator:</font></p>
<p></p>
<p><b><font face="Courier" size="12">-- Use Names</font></b><font face="Courier" size="12"> -- If checked, the list shows object names. Otherwise it shows IDs.</font></p>
<p></p>
<p><b><font face="Courier" size="12">-- Color Scripted Objects</font></b><font face="Courier" size="12"> -- If checked, objects in the list with scripts are shown in a different color (set on the prefs &gt; colors submenu -- see below). Slows list display slightly, so if you are on a slower machine or working with very long lists and want to maximize performance, uncheck this.</font></p>
<p></p>
<p><b><font face="Courier" size="12">-- Named Objects Only</font></b><font face="Courier" size="12"> -- If checked, the list shows only objects that have been given names. Excellent for filtering out labels, etc.</font></p>
<p></p>
<p><b><font face="Courier" size="12">-- Scripted Objects Only</font></b><font face="Courier" size="12"> -- If checked, the list shows only objects whose scripts are not empty. Works whether or not scripted objects are colored separately.</font></p>
<p></p>
<p><b><font face="Courier" size="12">-- Use Long Name or ID</font></b><font face="Courier" size="12"> -- If checked, shows the long name or id. Otherwise, the short name or abbreviated id.</font></p>
<p></p>
<p><b><font face="Courier" size="12">-- Show Tooltip</font></b><font face="Courier" size="12"> -- If checked, holding the pointer over an item in the list displays a tooltip with the long id of the item. Very annoying. ;-)</font></p>
<p></p>
<p><b><font face="Courier" size="12">-- Update List Automatically</font></b><font face="Courier" size="12"> -- If checked, the control list is updated whenever an object is created or deleted, whenever the current card changes, and whenever the selection changes it is checked. If it is unchecked, you must manually update the list -- see Update List Now below. Note that it is possible for the list to be out of sync even if this is checked. The list is updated in response to messages from the development environment, and there are some changes that will not trigger these messages. One example is if you change the name of a control using the Properties Contextual Menu. Many other things trigger a potential list update, so for example switching stacks should cause the list to update.</font></p>
<p></p>
<p><font face="Courier" size="12">----------</font></p>
<p></p>
<p><b><font face="Courier" size="12">-- Indent String...:</font></b><font face="Courier" size="12"> -- Navigator indents controls in the list based on the groups they are in. You can have Navigator use any string to do this. Use an empty string to prevent any indentation. The default "|  " provides for indentation and vertical lines showing the beginning and ending of groups. Three spaces makes for nice indentation with no vertical lines.</font></p>
<p></p>

<p><font face="Courier" size="12">Select any handler in the menu to edit the script of the object and display the handler selected. If the object has no script, select "Edit Script" to edit its script and get started.</font></p>
<p></p>
<p><b><font face="Courier" size="12">Call any handler</font></b><font face="Courier" size="12">: you can call any handler in the script by holding down the option (alt) key as you select a handler on the menu. An ask dialog will allow you to replace the parameter names with actual values. You can also cancel at that point -- this is a good way to copy a calling reference to a handler with all the arguments, for pasting into a handler.</font></p>
<p></p>
<p></p>
<p></p>
<p><b><font face="Courier" size="12">-- STACK MENU --</font></b></p>
<p><b><font face="Courier" size="12">----------------</font></b></p>
<p></p>
<p><font face="Courier" size="12">Allows you to select any user stack to examine with Navigator. The length of the menu is limited to the <b>Menu Limit</b> you set on the <b>Actions</b> menu. Use the Stack List to display more stacks.</font></p>
<p></p>
<p><b><font face="Courier" size="12">the topStack</font></b><font face="Courier" size="12"> -- The default "the topStack" makes it reflect the topStack always. </font></p>
<p></p>
<p><b><font face="Courier" size="12">Select a stack by pointing to it</font></b><font face="Courier" size="12">: hold down the Option/Alt key while selecting "the topStack"; continue to hold down the Option/Alt key and place the pointer over the stack you want to browse. Release the Option/Alt key to browse the controls in the stack.</font></p>
<p></p>
<p><b><font face="Courier" size="12">Stack List</font></b><font face="Courier" size="12"> -- Displays a list of your open stacks in the control list. Substacks are grouped with their mainStacks. Hold down the Option/Alt key to display a list of all stacks including development environment stacks.</font></p>
<p></p>
<p><font face="Courier" size="12">The menu is alphabetized by default; you can display the menu in the order returned by Revolution/MetaCard by holding down the <b>Shift</b> key while clicking the menu.</font></p>
<p></p>
<p><b><font face="Courier" size="12">Inspect all open stacks</font></b><font face="Courier" size="12"> by holding down the <b>Option</b> key when clicking the menu.</font></p>
<p></p>
<p><b><font face="Courier" size="12">Go to the stack selected</font></b><font face="Courier" size="12"> (instead of setting it as the stack to examine) by holding down the <b>Command</b> key as you select it.</font></p>
<p></p>
<p></p>
<p></p>
<p><b><font face="Courier" size="12">-- CARD MENU --</font></b></p>
<p><b><font face="Courier" size="12">---------------</font></b></p>
<p></p>
<p><font face="Courier" size="12">Allows you to select any card to examine with Navigator. The length of the menu is limited to the <b>Menu Limit</b> you set on the <b>Actions</b> menu. If you have a large number of cards (or groups), you can display them in the list instead of in the menu by selecting "card list" ("background list"). Then you can use the contextual menu to display controls for the card (or group).</font></p>
<p></p>
<p><font face="Courier" size="12">The default "this card" makes it reflect whichever card is showing in the stack.</font></p>
<p></p>
<p><font face="Courier" size="12">By default, the cards are presented in their order within the stack. You can alphabetize the card menu by holding down the <b>Shift</b> key while clicking the menu.</font></p>
<p></p>
<p><font face="Courier" size="12">The card menu shows backgrounds (groups) instead if you hold down the <b>Option</b> key. Only backgrounds with the backgroundBehavior property set to true are displayed. You can see a list of all the backgrounds/groups in the control list by selecting "background list" on the menu. You can then browse the controls of a particular group by control-clicking it and selecting "Browse Controls."</font></p>
<p></p>
<p><font face="Courier" size="12">Go to the card selected (instead of setting it as the card to examine) by holding down the <b>Command</b> key as you select it.</font></p>
<p></p>
<p></p>
<p></p>
<p><b><font face="Courier" size="12">-- COMMAND PANEL --</font></b></p>
<p><b><font face="Courier" size="12">-------------------</font></b></p>
<p></p>
<p><font face="Courier" size="12">Select <b>Command...</b> on the <b>Actions</b> menu or the contextual menu to display the <b>Command Panel</b>. At least one line must be hilited. Opening the Command Panel disables the control list. You have four buttons in the Command Panel: <b>Close</b>, <b>Set</b>, <b>Do, </b>and<b> Save</b>.</font></p>
<p></p>
<p><b><font face="Courier" size="12">CLOSE</font></b><font face="Courier" size="12"> -- closes the Command panel and enables the control list again.</font></p>
<p></p>
<p><b><font face="Courier" size="12">SET</font></b><font face="Courier" size="12"> -- takes the entries in the Command field and uses them to set properties on the selected objects. Any properties can be set to any values, including custom properties. The format for property/value pairs is this:</font></p>
<p></p>
<p><font face="Courier" size="12">propertyName=value</font></p>
<p></p>
<p><font face="Courier" size="12">No quotes are required. Multi-line values are allowed as long as they don't include an equals sign. You can set as many properties at a time as you like, so this would be fine:</font></p>
<p></p>
<p><font face="Courier" size="12">fillfore=red</font></p>
<p><font face="Courier" size="12">rect=40,40,90,90</font></p>
<p><font face="Courier" size="12">zMyCustomProperty=some value I use a lot</font></p>
<p><font face="Courier" size="12">to keep track of stuff.</font></p>
<p><font face="Courier" size="12">enabled=false</font></p>
<p></p>
<p><font face="Courier" size="12">In this example, note that "some value I use a lot to keep track of stuff" does <b>not</b> need to be quoted. If you quote it, the quotes will be included in the value of the property. Also note that it is multi-line. This command would change all the controls selected to red, make them the same size, disable them, and set a silly custom property on them.</font></p>
<p></p>
<p><b><font face="Courier" size="12">DO</font></b><font face="Courier" size="12"> -- runs the code in the Command field against each of the selected objects. This is extremely flexible. All looping and references are taken care of for you. This command is only available when using a licensed copy of Revolution -- it won't work in the Starter Kit.</font></p>
<p></p>
<p><font face="Courier" size="12">The following variables are available:</font></p>
<p></p>
<p><b><font face="Courier" size="12">tCount</font></b><font face="Courier" size="12"> -- the number of selected controls.</font></p>
<p><b><font face="Courier" size="12">tIndex</font></b><font face="Courier" size="12"> -- the number of the current control as the code executes on one control after another.</font></p>
<p><b><font face="Courier" size="12">tID</font></b><font face="Courier" size="12"> -- the long id of the current control as the code executes</font></p>
<p><b><font face="Courier" size="12">tIDList</font></b><font face="Courier" size="12"> -- An array of the long IDs of the selected controls. The index is the numbers 1, 2, 3...tCount.</font></p>
<p><b><font face="Courier" size="12">tOwnerID</font></b><font face="Courier" size="12"> -- the long id of the owner of tID (the current object).</font></p>
<p><b><font face="Courier" size="12">tCardID</font></b><font face="Courier" size="12"> -- the long id of the card that contains tID (the current object).</font></p>
<p><b><font face="Courier" size="12">tStackID</font></b><font face="Courier" size="12"> -- the long id of the stack that contains tID (the current object).</font></p>
<p><b><font face="Courier" size="12">tLastID</font></b><font face="Courier" size="12"> -- the long id of the previous control the code executed against. For the first control, it references the last.</font></p>
<p><b><font face="Courier" size="12">tNextID</font></b><font face="Courier" size="12"> -- the long id of the next control the code will execute against. For the last control, it references the first.</font></p>
<p></p>
<p><font face="Courier" size="12">Note: tLastID and tNextID wrap. For the first object in the list, tLastID will be the id of the last object in the list. For the last object in the list, tNextID will be the id of the first object in the list.</font></p>
<p></p>
<p><font face="Courier" size="12">Also, the defaultStack is set for each object to the stack that contains it, so for a series of cards this will work:</font></p>
<p></p>
<p><font face="Courier" size="12">place background "menubar1" onto tID</font></p>
<p></p>
<p><font face="Courier" size="12">Your code is wrapped in a try...end try, so an error on one object won't stop the code from working for other controls. That also means that your code can fail without letting you know.</font></p>
<p></p>
<p><font face="Courier" size="12">It is very useful to know that a long id in a variable can be used as a complete reference to an object. Here is an example of code you might run against a set of selected controls:</font></p>
<p></p>
<p><font face="Courier" size="12">set the loc of tID to 50,(tIndex*30)</font></p>
<p><font face="Courier" size="12">set the enabled of tID to not the enabled of tID</font></p>
<p><font face="Courier" size="12">set the name of tID to ("another radio button" &amp;&amp; tIndex)</font></p>
<p><font face="Courier" size="12">set zSomeCustomProp of tID to gSomeGlobalArray[tIndex]</font></p>
<p><font face="Courier" size="12">-- etc.</font></p>
<p></p>
<p><font face="Courier" size="12">When you are writing code to execute with the Do button, you can display a dialog to offer the user (yourself) a choice. Here's an example of how to do that:</font></p>
<p></p>
<p><font face="Courier" size="12">if tIndex is 1 then</font></p>
<p><font face="Courier" size="12">  setOptionLabel "Backgrounds:"</font></p>
<p><font face="Courier" size="12">  put the backgroundNames of this stack into tNames</font></p>
<p><font face="Courier" size="12">  setOptionListItems tNames</font></p>
<p><font face="Courier" size="12">  showOptions</font></p>
<p><font face="Courier" size="12">end if</font></p>
<p></p>
<p><font face="Courier" size="12">The first thing to notice is that the code is wrapped in an if statement. tIndex is the number of the loop iteration we're on. Putting the prompt code into an if based on "tIndex is 1" means this code will be executed only once no matter how many controls are selected. It will run the first time through, before the other loop code.</font></p>
<p></p>
<p><b><font face="Courier" size="12">setOptionLabel</font></b><font face="Courier" size="12"> sets a label in the dialog to whatever text you like. Use it as a prompt.</font></p>
<p></p>
<p><b><font face="Courier" size="12">setOptionListItems</font></b><font face="Courier" size="12"> will cause the prompt to display a list with the choices you give it. The list allows multiple selections.</font></p>
<p></p>
<p><b><font face="Courier" size="12">setOptionOneItems</font></b><font face="Courier" size="12"> will cause the prompt to display a popup menu with the items you provide.</font></p>
<p></p>
<p><b><font face="Courier" size="12">setOptionTwoItems</font></b><font face="Courier" size="12"> will cause the prompt to display a second popup menu with the items you provide.</font></p>
<p></p>
<p><b><font face="Courier" size="12">showOptions</font></b><font face="Courier" size="12"> actually displays the prompt. Execution of your code halts until the user (you) clicks OK or Cancel.</font></p>
<p></p>
<p><font face="Courier" size="12">When showOptions finishes, local variables will be set to the values chosen by the user (you) as follows:</font></p>
<p></p>
<p><b><font face="Courier" size="12">tOptionList</font></b><font face="Courier" size="12"> will be the items selected in the list, if you displayed the list. Each item is a separate line. You might use it like this:</font></p>
<p></p>
<p><font face="Courier" size="12">if word 1 of tID is "card" then </font></p>
<p><font face="Courier" size="12">  repeat for each line tBGName in tOptionList</font></p>
<p><font face="Courier" size="12">    place background tBGName onto tID</font></p>
<p><font face="Courier" size="12">  end repeat</font></p>
<p><font face="Courier" size="12">end if</font></p>
<p></p>
<p><b><font face="Courier" size="12">tOptionOne</font></b><font face="Courier" size="12"> is the choice made in the first popup if you showed it.</font></p>
<p></p>
<p></p>
<p><b><font face="Courier" size="12">tOptionTwo</font></b><font face="Courier" size="12"> is the choice made in the second popup if you showed it.</font></p>
<p></p>
<p><b><font face="Courier" size="12">SAVE</font></b><font face="Courier" size="12"> -- saves code in the Command field as a plugin. Navigator supports any number of plugins, which are just text files that store the code to run. Plugin commands are listed at the bottom of the contextual menu.</font></p>
<p></p>
