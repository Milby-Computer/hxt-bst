<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>The <font face="Courier" size="11">mouseMove</font><font face="helvetica" size="12"> message is sent to the object the mouse pointer is over, whenever the user moves the mouse. When the </font><font face="Courier" size="11">mouseMove</font><font face="helvetica" size="12"> message is sent to the field, it triggers this handler.</font></p>
<p></p>
<p>This example depends on the "link" text style, which creates grouped text. A piece of text with its <font face="Courier" size="11">textStyle</font><font face="helvetica" size="12"> set to "link" is treated as a single word by certain Transcript functions. This makes grouped text handy to use for hypertext or "clickable text" features. You can set text to the "link" style by selecting the text and choosing </font><b><font face="Lucida Grande" size="11">Text -&gt; Link,</font></b><font face="helvetica" size="12"> or by setting the chunk's </font><font face="Courier" size="11">textStyle</font><font face="helvetica" size="12"> property.</font></p>
<p></p>
<p><b><font face="Lucida Grande" size="11">Note: </font></b><font face="helvetica" size="12"> Revolution automatically underlines grouped text and changes its color. You can remove these appearance changes, which can be distracting in lengthy pieces of text, in the stack's Property Inspector.</font></p>
<p></p>
<p>The local declaration at the top of the script creates a variable called "storedHilitedChunk". Because it is declared outside any handler, "storedHilitedChunk" is a script local variable, meaning it's available to both handlers in this script. We'll see in a moment how this variable is used to store the location of the highlighted text, so it can be unhighlighted when the mouse pointer is over something else. For more details on script locals, see the section on <i><font face="Lucida Grande" size="11">Variables</font></i><font face="helvetica" size="12"> in the </font><i><font face="Lucida Grande" size="11">User's Guide</font></i><font face="helvetica" size="12">.</font></p>
<p></p>
<p>The first thing the <font face="Courier" size="11">mouseMove</font><font face="helvetica" size="12"> handler does is check to see whether there is already some highlighted text that needs to be unhighlighted. Then it goes on to highlight the text group the mouse is over (if any). It does this by setting the </font><font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> property of that text group to yellow. Of course, you can substitute any color.</font></p>
<p></p>
<p>The <font face="Courier" size="11">mouseChunk</font><font face="helvetica" size="12"> function returns a </font><i><font face="Lucida Grande" size="11">chunk expression</font></i><font face="helvetica" size="12"> describing the text's position in the field. For more details about chunk expressions, see the section on </font><i><font face="Lucida Grande" size="11">Using Chunk Expressions</font></i><font face="helvetica" size="12"> in the </font><i><font face="Lucida Grande" size="11">User's Guide</font></i><font face="helvetica" size="12">.  The </font><font face="Courier" size="11">mouseMove</font><font face="helvetica" size="12"> handler uses this function to find and manipulate the text that's under the mouse. If the </font><font face="Courier" size="11">mouseChunk</font><font face="helvetica" size="12"> returns empty, this means there is no text under the mouse, so the </font><font face="Courier" size="11">mouseMove</font><font face="helvetica" size="12"> handler doesn't need to highlight any text. If the </font><font face="Courier" size="11">textStyle</font><font face="helvetica" size="12"> of the </font><font face="Courier" size="11">mouseChunk</font><font face="helvetica" size="12"> does not include "link", this means the </font><font face="Courier" size="11">mouseChunk</font><font face="helvetica" size="12"> is not a text group, and therefore doesn't need to be highlighted. Only if the </font><font face="Courier" size="11">mouseChunk</font><font face="helvetica" size="12"> is not empty and its </font><font face="Courier" size="11">textStyle</font><font face="helvetica" size="12"> includes "link" does its </font><font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> need to be set.</font></p>
<p></p>
<p>As its last act, after setting the <font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> of the </font><font face="Courier" size="11">mouseChunk</font><font face="helvetica" size="12">, the </font><font face="Courier" size="11">mouseMove</font><font face="helvetica" size="12"> handler stores the </font><font face="Courier" size="11">mouseChunk</font><font face="helvetica" size="12"> in the "storedHilitedChunk" variable we declared at the top of the script. The script needs to remember what text it's highlighted, so it can unhighlight it when the mouse moves away from it.</font></p>
<p></p>
<p>Next time the mouse moves, the <font face="Courier" size="11">mouseMove</font><font face="helvetica" size="12"> handler checks this variable to see whether a text group needs to be unhighlighted. If the variable is empty, we haven't previously stored a chunk expression for highlighted text. If the variable isn't empty, but it's the same as the chunk the mouse is now over, then there's no need to unhighlight it. But if the variable has something in it, and it's </font><i><font face="Lucida Grande" size="11">not</font></i><font face="helvetica" size="12"> the same as the chunk the mouse is over, then the mouse is no longer over the previously-highlighted chunk of text and we need to unhighlight it. The handler removes the highlighting by setting the chunk's </font><font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> property to empty. This allows the field's own background color to show, so the chunk's background is no longer different from the surrounding text.</font></p>
<p></p>
<p>The <font face="Courier" size="11">mouseLeave</font><font face="helvetica" size="12"> handler, which is triggered when the mouse moves outside the field, simply cleans up by making sure any highlighted text gets unhighlighted. It does this in the same way the </font><font face="Courier" size="11">mouseMove</font><font face="helvetica" size="12"> handler does it, by checking the "storedHilitedChunk" variable and setting the chunk's </font><font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> to empty if necessary. If the mouse is moving slowly enough, this handler isn't needed, because when the mouse moves off the highlighted text on the way to the edge of the field, it triggers the </font><font face="Courier" size="11">mouseMove</font><font face="helvetica" size="12"> handler, which unhighlights the text. However, if the mouse moves fast enough, it may reach the edge of the field before a </font><font face="Courier" size="11">mouseMove</font><font face="helvetica" size="12"> message is sent. In this case, the </font><font face="Courier" size="11">mouseLeave</font><font face="helvetica" size="12"> handler ensures that any leftover highlighting is cleaned up.</font></p>
