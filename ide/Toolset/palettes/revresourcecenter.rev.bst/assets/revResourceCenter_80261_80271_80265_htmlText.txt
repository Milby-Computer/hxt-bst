<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>The <font face="Courier" size="11">rawKeyDown</font><font face="helvetica" size="12"> message is sent whenever a character key is pressed (and whenever a mouse scroll wheel is used). If the blinking text cursor is in a field, the </font><font face="Courier" size="11">rawKeyDown</font><font face="helvetica" size="12"> message is sent to that field. When the </font><font face="Courier" size="11">rawKeyDown</font><font face="helvetica" size="12"> message is sent to the field, it triggers this handler.</font></p>
<p></p>
<p>In this example, the <font face="Courier" size="11">rawKeyDown</font><font face="helvetica" size="12"> handler should be placed in the script of a field. The handler checks the contents of the field every time a key is pressed, and adds a scrollbar if the field's text becomes too large to fit. If the field already has a scrollbar and the user removes some text, the scrollbar is removed.</font></p>
<p></p>
<p><b><font face="Lucida Grande" size="11">Note:</font></b><font face="helvetica" size="12">  We use a </font><font face="Courier" size="11">rawKeyDown</font><font face="helvetica" size="12"> handler instead of a </font><font face="Courier" size="11">keyDown</font><font face="helvetica" size="12"> handler because </font><font face="Courier" size="11">rawKeyDown</font><font face="helvetica" size="12"> recognizes even keystrokes that don't correspond to a character. For example, pressing the Delete key sends a </font><font face="Courier" size="11">rawKeyDown</font><font face="helvetica" size="12"> message, but not a </font><font face="Courier" size="11">keyDown</font><font face="helvetica" size="12"> message.</font></p>
<p></p>
<p>The <font face="Courier" size="11">rawKeyDown</font><font face="helvetica" size="12"> message passes a parameter, which is the keycode of the key that was pressed. In this example, however, we don't really care which key was pressed--we only need to know that the user typed a key, so we can check the height of the text--so we haven't used this parameter or assigned a name to it.</font></p>
<p></p>
<p>This <font face="Courier" size="11">rawKeyDown</font><font face="helvetica" size="12"> handler checks the </font><font face="Courier" size="11">formattedHeight</font><font face="helvetica" size="12"> property of the field, which is the total height in pixels that the text would occupy if it were all displayed at once. If the </font><font face="Courier" size="11">formattedHeight</font><font face="helvetica" size="12"> is larger than the field's height, the text is too large to fit completely within the visible area of the field. If the text won't fit, the handler shows a vertical scrollbar by setting the field's </font><font face="Courier" size="11">vScrollbar</font><font face="helvetica" size="12"> property to true. If the text does fit, the handler sets the </font><font face="Courier" size="11">vScrollbar</font><font face="helvetica" size="12"> to false.</font></p>
<p></p>
<p>Finally, the handler uses the <font face="Courier" size="11">pass</font><font face="helvetica" size="12"> control structure to let the </font><font face="Courier" size="11">rawKeyDown</font><font face="helvetica" size="12"> message proceed along the message path. If the </font><font face="Courier" size="11">rawKeyDown</font><font face="helvetica" size="12"> message is trapped, the keystroke is not processed, so it's important to pass the message if you want the keystroke to appear. If you don't pass the </font><font face="Courier" size="11">rawKeyDown</font><font face="helvetica" size="12"> message, you won't be able to type anything into the field.</font></p>
<p></p>
<p><font color="#404040"><img src="202651" char=" "> Tip:</font> To save lines of code, we can rewrite the conditional in this handler like this:</p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">set</font><font face="Courier" size="11"> the </font><font face="Courier" size="11" color="#FF0000">vScrollbar</font><font face="Courier" size="11"> of </font><font face="Courier" size="11" color="#F88017">me</font><font face="Courier" size="11"> to \</font></p>
<p><font face="Courier" size="11">     (the </font><font face="Courier" size="11" color="#FF0000">formattedHeight</font><font face="Courier" size="11"> of </font><font face="Courier" size="11" color="#F88017">me</font><font face="Courier" size="11"> &gt; the </font><font face="Courier" size="11" color="#FF0000">height</font><font face="Courier" size="11"> of </font><font face="Courier" size="11" color="#F88017">me</font><font face="Courier" size="11">)</font></p>
<p></p>
<p>This does the same thing as the if control structure in the original handler: if the <font face="Courier" size="11">formattedHeight</font><font face="helvetica" size="12"> is greater than the </font><font face="Courier" size="11">height</font><font face="helvetica" size="12">, the expression (</font><font face="Courier" size="11">the formattedHeight of me &gt; the height of me</font><font face="helvetica" size="12">) is true, so the </font><font face="Courier" size="11">vScrollbar</font><font face="helvetica" size="12"> is set to </font><font face="Courier" size="11">true</font><font face="helvetica" size="12">. This is a little harder to read, but saves a line. You can use this same technique to transform any conditional that sets a property to </font><font face="Courier" size="11">true</font><font face="helvetica" size="12"> or </font><font face="Courier" size="11">false</font><font face="helvetica" size="12">.</font></p>
