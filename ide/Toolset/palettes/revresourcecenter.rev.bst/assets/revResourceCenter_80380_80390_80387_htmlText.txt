<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>The <font face="Courier" size="11">keyDown</font><font face="helvetica" size="12"> message is sent whenever a character key is pressed. If the blinking text cursor is in a field, the </font><font face="Courier" size="11">keyDown</font><font face="helvetica" size="12"> message is sent to that field. When the </font><font face="Courier" size="11">keyDown</font><font face="helvetica" size="12"> message is sent to the field, it triggers this handler.</font></p>
<p></p>
<p>The parameter, which we call "theKey" in this example, is the character that was pressed. For example, if you press the "A" key, the <font face="Courier" size="11">keyDown</font><font face="helvetica" size="12"> message that is sent has "A" as its parameter. This means we can simply check whether the parameter is a number to find out whether to accept the keypress or not.</font></p>
<p></p>
<p>The <font face="Courier" size="11">is a</font><font face="helvetica" size="12"> operator can accept strings that contain non-digits as numbers. For example, "22.5" is a number even though it contains a period. However, the parameter to the keyDown message is always a single character, and for a single character to be a number, it must be a digit from 0 to 9.</font></p>
<p></p>
<p>The Revolution engine must receive the <font face="Courier" size="11">keyDown</font><font face="helvetica" size="12"> message in order to respond to a typed character (or in other words, the entry of a character is triggered by the </font><font face="Courier" size="11">keyDown</font><font face="helvetica" size="12"> message), so trapping the message and not passing it prevents typing from being entered in the field.</font></p>
<p></p>
<p>If the character that was pressed isn't a number, the <font face="Courier" size="11">keyDown</font><font face="helvetica" size="12"> handler beeps to alert the user to the mistake, and doesn't </font><font face="Courier" size="11">pass</font><font face="helvetica" size="12"> the message on, preventing the character from appearing in the field. If the character is a number, the handler uses the </font><font face="Courier" size="11">pass</font><font face="helvetica" size="12"> control structure to allow the keyDown message to proceed along the message path.  For more details on the message path, see the section </font><i><font face="Lucida Grande" size="11">The Message Path</font></i><font face="helvetica" size="12"> in the </font><i><font face="Lucida Grande" size="11">User's Guide</font></i><font face="helvetica" size="12">.</font></p>
