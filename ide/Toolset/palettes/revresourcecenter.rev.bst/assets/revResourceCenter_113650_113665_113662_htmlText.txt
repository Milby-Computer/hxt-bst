<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>This custom function is called with a statement such as the following:</p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">put</font><font face="Courier" size="11"> commonLines(baseList,field "New Lines") into myLines</font></p>
<p></p>
<p>The "commonLines" function finds the lines that are in both of two different lists. It does this by creating an <i><font face="Lucida Grande" size="11">array</font></i><font face="helvetica" size="12"> for each list, with the </font><font face="Courier" size="11">keys</font><font face="helvetica" size="12"> being the text of the unique lines, then using the </font><font face="Courier" size="11">intersect</font><font face="helvetica" size="12"> command to find the </font><font face="Courier" size="11">keys</font><font face="helvetica" size="12"> that are in both arrays.</font></p>
<p></p>
<p>The first thing the handler does is to create an array of the lines in the firstList parameter. You might think that we could use the <font face="Courier" size="11">combine</font><font face="helvetica" size="12"> command to do this, but the </font><font face="Courier" size="11">combine</font><font face="helvetica" size="12"> command would make each line into the content of an array element. Instead, we want each line to be the </font><font face="Courier" size="11">key</font><font face="helvetica" size="12"> of an element. The content of each element doesn't matter for this handler. This example puts true into each element; we could actually put any value into each element, and the handler would work the same way.</font></p>
<p></p>
<p>To create this array, we use the <font face="Courier" size="11">repeat</font><font face="helvetica" size="12"> control structure to make an element for each line in the firstList. The element's key is the text in the line, and its content is the true constant. If a line appears more than once in the list, the array will contain only one element for that line, instead of one for each repetition, because an array can't have two elements with the same key. But this is fine: since the function is looking for common lines, we don't care whether a line is repeated in the firstList--only whether it appears at all.</font></p>
<p></p>
<p>Next, the handler creates an array from the secondList parameter, in the same way as it used the firstList. We now have two arrays, and the keys of each array are the lines in the corresponding list. To find the common lines, we need a list of all the elements whose keys appear in both arrays. This is exactly what the <font face="Courier" size="11">intersect</font><font face="helvetica" size="12"> command does: given two arrays, it retains only the elements whose keys match.</font></p>
<p></p>
<p>After the <font face="Courier" size="11">intersect</font><font face="helvetica" size="12"> command is executed, the "firstArray" variable contains only the elements whose keys are in both arrays. Each key is the text of a common line, so the function returns the list of keys in this array.</font></p>
<p></p>
<p>This handler takes advantage of Transcript's ability to create and work with associative arrays. Most languages that support arrays only let you use integers as the keys, but in Transcript, you can use <i><font face="Lucida Grande" size="11">any string</font></i><font face="helvetica" size="12"> as a key. In this example, the capability lets us use the keys of the array--not just the contents-- as meaningful content, and this technique using the </font><font face="Courier" size="11">intersect</font><font face="helvetica" size="12"> command would not be possible without the use of associative arrays.</font></p>
<p></p>
<p><b><font face="Lucida Grande" size="11">A note on efficiency:</font></b></p>
<p>You might wonder whether the array operations in this example--constructing the two arrays and executing the intersect command--make it slower than an example, like the following, that uses chunk expression to find out whether each line in the firstList is also in the secondList or not:</p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">repeat</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#980517">for</font><font face="Courier" size="11"> each line thisLine in firstList</font></p>
<p><font face="Courier" size="11">    </font><font face="Courier" size="11" color="#980517">if</font><font face="Courier" size="11"> thisLine is among the lines of secondList</font></p>
<p><font face="Courier" size="11">    </font><font face="Courier" size="11" color="#980517">then</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#0000FF">put</font><font face="Courier" size="11"> thisLine &amp; </font><font face="Courier" size="11" color="#0000FF">return</font><font face="Courier" size="11"> after theCommonLines</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">end</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#980517">repeat</font></p>
<p></p>
<p>While array operations typically have some overhead, it turns out that the approach used in this example is much faster than the one illustrated above if the lists are longer than ten lines or so. If the firstList is ten lines long, the two approaches are approximately equal (within a factor of two), but as the firstList grows in length, the approach using arrays quickly gains a speed advantage.</p>
