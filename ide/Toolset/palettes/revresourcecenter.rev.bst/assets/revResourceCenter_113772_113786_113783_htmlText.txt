<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>This custom function is called with a statement such as the following:</p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">sort</font><font face="Courier" size="11"> lines of field "Titles" by withoutLeadingArticle(each)</font></p>
<p></p>
<p>The custom function works in tandem with the <font face="Courier" size="11">sort</font><font face="helvetica" size="12"> container command. When you </font><font face="Courier" size="11">sort</font><font face="helvetica" size="12"> the lines of a container, the </font><font face="Courier" size="11">each</font><font face="helvetica" size="12"> keyword serves as a </font><i><font face="Lucida Grande" size="11">placeholder</font></i><font face="helvetica" size="12"> for each line in the container. In the above statement, each line is passed to the "withoutLeadingArticle" function, and the returned value is used as the sort key for that line. The "withoutLeadingArticle" function simply removes any leading articles (the words "A", "An", and "The"), leaving the rest of the title to be used as the sort key.</font></p>
<p></p>
<p>The "withoutLeadingArticle" function checks the first word of the title that's passed to it. If this word is an article ("the", "an", or "a"), it deletes the word; otherwise, the title is left alone. The function then returns the (possibly modified) title. This modified title, without any leading articles, is used as the sort key for that title. This means that the sort container command behaves as though it's sorting by the modified titles, instead of the titles themselves. The parameter, "theTitle", is used as a variable in the function; the function changes the parameter directly.</p>
<p></p>
<p>Using a custom function with the <font face="Courier" size="11">each</font><font face="helvetica" size="12"> keyword makes the </font><font face="Courier" size="11">sort</font><font face="helvetica" size="12"> container command much more flexible, because you can use the custom function to create any custom sort criteria you want to use. You can eliminate certain parts of the lines being sorted from consideration, as we've done here. Or you can use another custom function to transform lines in other ways.</font></p>
<p></p>
<p>For example, you could write a custom function for international sorting, which would change accented characters like &eacute; or &uuml; into their ASCII equivalents (e and u). A sort done with this custom function would sort accented characters as though they were the unaccented equivalents, instead of sorting all accented characters after the end of the unaccented alphabet.</p>
<p></p>
