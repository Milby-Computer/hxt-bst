<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>This custom command is used with a statement like the following:</p>
<p></p>
<p><font face="Courier" size="11"> convertChar "ASCII"</font></p>
<p><font face="Courier" size="11"> convertChar</font></p>
<p><font face="Courier" size="11"> convertChar "character"</font></p>
<p></p>
<p>You might call it from a button's <font face="Courier" size="11">mouseUp</font><font face="helvetica" size="12"> handler. This handler assumes there is a field called "Character" and one called "ASCII", which hold the character and its ASCII equivalent.</font></p>
<p></p>
<p>Unlike some languages (such as C) and like others (such as Perl), Transcript treats individual characters as strings, not as numbers. This means that you can't simply add a number to a character to change its ASCII value. You must first convert the character to its ASCII value. In Transcript, this is done with the <font face="Courier" size="11">charToNum</font><font face="helvetica" size="12"> function. Converting a number to the corresponding character is done with the </font><font face="Courier" size="11">numToChar</font><font face="helvetica" size="12"> function.</font></p>
<p></p>
<p>The core of the handler consists of these two statements:</p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">put</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#F88017">numToChar</font><font face="Courier" size="11">(it) into field "Character"</font></p>
<p>and</p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">put</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#F88017">charToNum</font><font face="Courier" size="11">(field "Character") into field "ASCII"</font></p>
<p>The first of these statements converts a number to a character; the second converts a character to its ASCII value.</p>
<p></p>
<p>The handler takes one parameter, which is what's being converted (either "character" or "ASCII"). However, you can call the handler without this parameter, and the handler itself will determine which direction to convert it. It's possible to do this because the first part of the handler checks to see whether the parameter has been specified. If not, the parameter will be empty. In this case, if the "Character" field has something in it, the handler assumes you want to convert it to the ASCII value. Otherwise, it assumes you want to convert a number to the corresponding character.</p>
<p></p>
<p>You can use this method with any handler to assign a default value to one or more of the parameters. Just check whether the parameter is empty. If it is, then no value has been passed, and you set the default value by simply putting the desired default into the parameter.</p>
<p></p>
<p>Once it's figured out what conversion type to use, the handler makes sure there's something in the field to convert. If not, it uses the return control structure to pass an error message back to the calling handler. Since this is a message handler, the returned value is placed in the result function. When using the "convertChar" custom command, a handler can check the result to find out whether there was an error:</p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">on</font><font face="Courier" size="11"> mouseUp</font></p>
<p><font face="Courier" size="11">    convertChar</font></p>
<p><font face="Courier" size="11">    </font><font face="Courier" size="11" color="#980517">if</font><font face="Courier" size="11"> the </font><font face="Courier" size="11" color="#F88017">result</font><font face="Courier" size="11"> is not empty </font><font face="Courier" size="11" color="#980517">then</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#0000FF">beep</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#68228B">-- or display the error</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">end</font><font face="Courier" size="11"> mouseUp</font></p>
<p></p>
<p>Displaying a character's ASCII value is simple: the handler just puts the resulting number into the "ASCII" field. However, displaying an ASCII value's corresponding character is a little more complicated, because some characters (such as the backspace character) aren't printable--that is, they can't be displayed in a field or on the screen. In preparing this example, the button that contains the script was given a custom property called "asciiChart" that contains the standard 3-letter codes for the non-printable characters. (You could also write a custom function that would return the 3-letter code given the ASCII value.) The handler uses this custom property to display these codes. (If you want to accept only printable characters, you can modify the handler so that instead of checking the custom property, it displays an error message if the ASCII value indicates a non-printable character.) If the character is printable, the handler puts it into the "Character" field.</p>
<p></p>
<p>It is often convenient, as in this example, to get or change a field's contents directly in a utility handler. (Working with a field is significantly slower than working with a parameter or variable. But in this example, you don't need to refer to the field often enough to really make a difference.) The example could be made more general simply by replacing the field references with a parameter, as in the following fragment:</p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">on</font><font face="Courier" size="11"> convertChar theSource,whatToConvert </font><font face="Courier" size="11" color="#68228B">-- new parameter</font></p>
<p><font face="Courier" size="11">    </font><font face="Courier" size="11" color="#980517">if</font><font face="Courier" size="11"> theSource is "character" </font><font face="Courier" size="11" color="#980517">then</font></p>
<p><font face="Courier" size="11">      </font><font face="Courier" size="11" color="#0000FF">put</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#F88017">charToNum</font><font face="Courier" size="11">(char 1 of whatToConvert) into field "ASCII"</font></p>
<p><font face="Courier" size="11">    </font><font face="Courier" size="11" color="#980517">else</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#68228B">-- convert ASCII value to character</font></p>
<p><font face="Courier" size="11">    </font><font face="Courier" size="11" color="#68228B">-- etc. ...</font></p>
