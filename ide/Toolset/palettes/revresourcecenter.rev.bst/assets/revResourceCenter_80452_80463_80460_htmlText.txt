<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>This custom function is called with a statement such as one of the following:</p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">get</font><font face="Courier" size="11"> RGBFromColorName("LightGreen")</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">put</font><font face="Courier" size="11"> RGBFromColorName(the </font><font face="Courier" size="11" color="#FF0000">borderColor</font><font face="Courier" size="11"> of </font><font face="Courier" size="11" color="#F88017">me</font><font face="Courier" size="11">) into testColor</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">set</font><font face="Courier" size="11"> the </font><font face="Courier" size="11" color="#FF0000">textColor</font><font face="Courier" size="11"> of field 1 to RGBFromColorName(theColor)</font></p>
<p></p>
<p>This handler depends on a subtle trick: setting the <font face="Courier" size="11">backgroundPixel</font><font face="helvetica" size="12"> of an object to itself transforms the object's </font><font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> to its RGB equivalent. This also works for the other seven pixel properties and their color equivalents. To accomplish this, the handler creates a temporary button and sets its </font><font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> to the color name. Then it sets the button's </font><font face="Courier" size="11">backgroundPixel</font><font face="helvetica" size="12"> to itself. The result is that the button's </font><font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> is transformed into the RGB equivalent.</font></p>
<p></p>
<p>The first thing the handler does is to use the is a operator to check whether the theColor parameter is a color or not. The <font face="Courier" size="11">is a</font><font face="helvetica" size="12"> operator also allows the parameter to be an RGB triplet or HTML-style color reference. Passing such a color reference to this function isn't very useful. But it won't cause an execution error, and the result is reasonable--if theColor is an RGB triplet, the function just returns that triplet--so we don't try to make sure that theColor is a color name, just that it's some sort of valid color reference. If theColor is not a color reference, the handler uses the </font><font face="Courier" size="11">return</font><font face="helvetica" size="12"> control structure to stop the handler.</font></p>
<p></p>
<p>Next, the handler creates a button. The <font face="Courier" size="11">create invisible</font><font face="helvetica" size="12"> form of the </font><font face="Courier" size="11">create</font><font face="helvetica" size="12"> command makes sure the button is hidden. This way, the user doesn't see the button on the screen while the handler executes. You could also create the button outside the window's rectangle, or use the </font><font face="Courier" size="11">lock screen</font><font face="helvetica" size="12"> command to prevent the button from being seen. But using the create invisible form is simpler. If </font><font face="Courier" size="11">the result</font><font face="helvetica" size="12"> function is set, the button couldn't be created for some reason, so the handler uses the </font><font face="Courier" size="11">return</font><font face="helvetica" size="12"> control structure to stop it and return an error message.</font></p>
<p></p>
<p>The handler sets the button's <font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> to the color name theColor, then sets the button's </font><font face="Courier" size="11">backgroundPixel</font><font face="helvetica" size="12"> property to itself:</font></p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">set</font><font face="Courier" size="11"> the </font><font face="Courier" size="11" color="#FF0000">backgroundPixel</font><font face="Courier" size="11"> of last button to \</font></p>
<p><font face="Courier" size="11">     the </font><font face="Courier" size="11" color="#FF0000">backgroundPixel</font><font face="Courier" size="11"> of last button</font></p>
<p></p>
<p>Because the button is the most recently-created control on the card, and we haven't changed its layer since it was created, it is also the topmost control on the card. This makes it the <font face="Courier" size="11">last</font><font face="helvetica" size="12"> button, so we can refer to it that way when setting its color.</font></p>
<p></p>
<p>The button's color is unchanged, but its <font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> property is now in RGB triplet form, instead of being a color name. Now we just need to clean up: the handler gets the button's </font><font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> (which places it in the it variable), deletes the button, switches back to the Browse tool, and returns the </font><font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12">.</font></p>
