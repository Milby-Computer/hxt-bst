<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>This custom function is called with a statement such as one of the following:</p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">get</font><font face="Courier" size="11"> truncatedToFit("My </font><font face="Courier" size="11" color="#FF0000">Name</font><font face="Courier" size="11">",120)</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">put</font><font face="Courier" size="11"> truncatedToFit(it,the </font><font face="Courier" size="11" color="#FF0000">width</font><font face="Courier" size="11"> of field "Answer" - 8) \</font></p>
<p><font face="Courier" size="11">    into field "Answer"</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">if</font><font face="Courier" size="11"> truncatedToFit(myVariable,250) is it </font><font face="Courier" size="11" color="#980517">then</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#0000FF">next</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#980517">repeat</font></p>
<p></p>
<p>The function operates by using <font face="Courier" size="11">the formattedText</font><font face="helvetica" size="12"> property of fields. The </font><font face="Courier" size="11">formattedText</font><font face="helvetica" size="12"> property tells you where line breaks appear when the text is wrapped in a field. Since it's a field property, it requires you to put the text into a field. Because of this, the handler needs to create a temporary field for the purpose. To prevent the user from seeing this field, the handler uses the create invisible form.</font></p>
<p></p>
<p>Next, the handler sets the width of the new field to the pixel width passed to it in the second parameter. To refer to this new field, the handler uses the expression last field, which means the field with the highest <font face="Courier" size="11">layer</font><font face="helvetica" size="12"> property (in front of any other fields). Because controls are always created on top of any existing controls, a newly-created control's layer is higher than any other control's, so unless you change the layer, a new control is always the last control.</font></p>
<p></p>
<p>When text is put into a field, the text is word-wrapped to the field's width. The <font face="Courier" size="11">formattedText</font><font face="helvetica" size="12"> property returns the text with line breaks inserted where the text wraps to the next line. The first line of the </font><font face="Courier" size="11">formattedText</font><font face="helvetica" size="12">, then, is the first (physical) line of the wrapped text. This is the amount of the text that fits into the width of the field, and this is the value that will be returned by the custom function.</font></p>
<p></p>
<p><img src="202653" char=" "> <b><font face="Lucida Grande" size="11">Important!</font></b><font face="helvetica" size="12"> The </font><font face="Courier" size="11">formattedText</font><font face="helvetica" size="12"> property works only if the field's </font><font face="Courier" size="11">dontWrap</font><font face="helvetica" size="12"> property is set to </font><font face="Courier" size="11">false</font><font face="helvetica" size="12">; otherwise the text won't wrap. Normally, the </font><font face="Courier" size="11">dontWrap</font><font face="helvetica" size="12"> of a newly-created field is </font><font face="Courier" size="11">false</font><font face="helvetica" size="12"> by default, so you don't need to change this.</font></p>
<p></p>
<p>Finally, the function must clean up after itself before returning its value. It deletes the field, then sets the tool back to the Browse tool. (Creating the field automatically switched to the Pointer tool.)</p>
<p></p>
<p>Several variations on this handler are possible. For example, you may want to specify the font, size, or style of the text. You can do this by setting the field's <font face="Courier" size="11">textFont</font><font face="helvetica" size="12">, </font><font face="Courier" size="11">textSize</font><font face="helvetica" size="12">, and </font><font face="Courier" size="11">textStyle</font><font face="helvetica" size="12"> properties.</font></p>
<p></p>
<p>Another variation is to check whether the returned text is the same as the original text (that is, whether the text is short enough to fit in the specified width), and, if not, add an ellipsis to the end of the text to indicate that it's not complete. To do this, you place this line in the function handler, just before the return statement:</p>
<p></p>
