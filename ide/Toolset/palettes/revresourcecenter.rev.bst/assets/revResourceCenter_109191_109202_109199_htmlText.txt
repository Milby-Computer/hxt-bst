<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>The <font face="Courier" size="11">returnInField</font><font face="helvetica" size="12"> message is sent when the Return key is pressed while the blinking text cursor is in the field. When the </font><font face="Courier" size="11">returnInField</font><font face="helvetica" size="12"> message is sent to the field, it triggers this handler.</font></p>
<p></p>
<p>The keyword <font face="Courier" size="11">me</font><font face="helvetica" size="12"> refers to the current object: the object whose script contains the handler that is executing. In this example, </font><font face="Courier" size="11">me</font><font face="helvetica" size="12"> is a reference to the field, because the </font><font face="Courier" size="11">returnInField</font><font face="helvetica" size="12"> handler is in the field's script. If the field is </font><font face="Courier" size="11">empty</font><font face="helvetica" size="12">, there is nothing to find, so the handler just beeps. Otherwise, the handler uses the </font><font face="Courier" size="11">find</font><font face="helvetica" size="12"> command to search for whatever text is in the field.</font></p>
<p></p>
<p>The <font face="Courier" size="11">find</font><font face="helvetica" size="12"> command is a simple search. By default, it finds and highlights the next instance where all the words in the field appear. It starts searching on the current card, then continues to search other cards in the current stack.  The default form of the search doesn't require all the search words to appear in the same order as specified, or even in the same field, as long as they are all found on the same card. You can use several other search options (see the </font><font face="Courier" size="11">find</font><font face="helvetica" size="12"> entry in the </font><i><font face="Lucida Grande" size="11">Transcript Dictionary</font></i><font face="helvetica" size="12"> for more information). Here are a few useful variations:</font></p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">find</font><font face="Courier" size="11"> string the </font><font face="Courier" size="11" color="#FF0000">text</font><font face="Courier" size="11"> of </font><font face="Courier" size="11" color="#F88017">me</font></p>
<p><font face="Courier" size="11">     </font><font face="Courier" size="11" color="#68228B">-- find the entire contents as a whole phrase</font></p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">find</font><font face="Courier" size="11"> the </font><font face="Courier" size="11" color="#FF0000">text</font><font face="Courier" size="11"> of </font><font face="Courier" size="11" color="#F88017">me</font><font face="Courier" size="11"> in field "Topic"</font></p>
<p><font face="Courier" size="11">     </font><font face="Courier" size="11" color="#68228B">-- find in a particular field - most useful for</font></p>
<p><font face="Courier" size="11">     </font><font face="Courier" size="11" color="#68228B">-- fields in a group that's placed on multiple cards</font></p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">find</font><font face="Courier" size="11"> words the </font><font face="Courier" size="11" color="#FF0000">text</font><font face="Courier" size="11"> of </font><font face="Courier" size="11" color="#F88017">me</font></p>
<p><font face="Courier" size="11">     </font><font face="Courier" size="11" color="#68228B">-- requires each word in the field to be a whole</font></p>
<p><font face="Courier" size="11">     </font><font face="Courier" size="11" color="#68228B">-- word, so searching for "sea" won't find "seal"</font></p>
<p></p>
<p><font color="#404040"><img src="202653" char=" "> </font><b><font face="Lucida Grande" size="11">Important!</font></b><font face="helvetica" size="12">  There's one last piece of the puzzle to be aware of:  if the Search field is in the stack to be searched, the field's </font><font face="Courier" size="11">dontSearch</font><font face="helvetica" size="12"> property must be set to true. Otherwise, it will try to find the search text in itself. (Of course, it will succeed, since the Search field does contain the search text, so it will never find it anywhere else.)</font></p>
<p></p>
<p>Finally, suppose you want the Search field to look in another stack. For example, the Search box may be in a separate Search palette window, and you want to search some other "document" window. The <font face="Courier" size="11">find</font><font face="helvetica" size="12"> command operates on the current stack, so if you set the </font><font face="Courier" size="11">defaultStack</font><font face="helvetica" size="12"> property to the stack you want to search, the Search field will look in that stack instead:</font></p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">set</font><font face="Courier" size="11"> the </font><font face="Courier" size="11" color="#FF0000">defaultStack</font><font face="Courier" size="11"> to "My Records Window"</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">find</font><font face="Courier" size="11"> the </font><font face="Courier" size="11" color="#FF0000">text</font><font face="Courier" size="11"> of </font><font face="Courier" size="11" color="#F88017">me</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#68228B">-- searches in "My Records Window"</font></p>
<p></p>
<p><font color="#404040"><img src="202651" char=" "> </font><b><font face="Lucida Grande" size="11">Tip:</font></b><font face="helvetica" size="12"> If your Search field is contained in a palette, to search the front most editable stack:</font></p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">set</font><font face="Courier" size="11"> the </font><font face="Courier" size="11" color="#FF0000">defaultStack</font><font face="Courier" size="11"> to the </font><font face="Courier" size="11" color="#F88017">topStack</font></p>
<p></p>
<p>If there are no editable stack windows open, it searches modeless dialog boxes (like the Documentation windows).</p>
<p></p>
<p><b><font face="Lucida Grande" size="11">Note:</font></b><font face="helvetica" size="12"> The find command is only one of a number of ways to search text in Revolution.  It is the simplest for this purpose because it cycles through fields and cards in order and automatically highlights the results.  However to have complete control over the search, to search a variable, database or another container there are a number of other methods avaliable.  See the section on </font><i><font face="Lucida Grande" size="11">Processing Text and Data</font></i><font face="helvetica" size="12"> in the </font><i><font face="Lucida Grande" size="11">User's Guide</font></i><font face="helvetica" size="12"> for more details.</font></p>
