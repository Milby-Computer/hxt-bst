<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>This custom function is called with a statement such as one of the following:</p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">get</font><font face="Courier" size="11"> titledWindowIsOpen("Untitled 2")</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">put</font><font face="Courier" size="11"> titledWindowIsOpen(the </font><font face="Courier" size="11" color="#F88017">selectedText</font><font face="Courier" size="11">) into isOpen</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">set</font><font face="Courier" size="11"> the </font><font face="Courier" size="11" color="#FF0000">hilite</font><font face="Courier" size="11"> of button "Open" to titledWindowIsOpen(it)</font></p>
<p></p>
<p>The "titledWindowIsOpen" function checks whether a stack window is open, but using its <font face="Courier" size="11">label</font><font face="helvetica" size="12"> property instead of its name. (Usually, you use the </font><font face="Courier" size="11">name</font><font face="helvetica" size="12"> of a stack instead, because it's unique--only one stack with a particular name can be open at a time, while several open stacks might have the same </font><font face="Courier" size="11">label</font><font face="helvetica" size="12">.) If there is an open stack window with the specified title, the function returns </font><font face="Courier" size="11">true</font><font face="helvetica" size="12">; if there is no such window, it returns </font><font face="Courier" size="11">false</font><font face="helvetica" size="12">.</font></p>
<p></p>
<p>First, the handler gets a list of all open stacks, using the openStacks function, and puts it into a variable called "openStacksList".</p>
<p></p>
<p>The <font face="Courier" size="11">openStacks</font><font face="helvetica" size="12"> function returns the </font><font face="Courier" size="11">name</font><font face="helvetica" size="12"> of stack windows, not their </font><font face="Courier" size="11">label</font><font face="helvetica" size="12"> so to find out whether there is an open window with a certain </font><font face="Courier" size="11">title</font><font face="helvetica" size="12">, we need to check the </font><font face="Courier" size="11">label</font><font face="helvetica" size="12"> property of each stack returned by </font><font face="Courier" size="11">the openStacks</font><font face="helvetica" size="12">. This is done in a </font><font face="Courier" size="11">repeat</font><font face="helvetica" size="12"> loop. The loop goes through each of the lines in the "openStacksList" variable and checks each stack's label property. If the label matches the window title we're looking for, the function returns </font><font face="Courier" size="11">true</font><font face="helvetica" size="12">.</font></p>
<p></p>
<p>The <font face="Courier" size="11">return</font><font face="helvetica" size="12"> control structure halts the handler it's in, so as soon as we find a stack with the right title, the handler ends. If the handler checks all the open stacks and doesn't find one with the right title, it returns </font><font face="Courier" size="11">false</font><font face="helvetica" size="12">.</font></p>
<p></p>
<p><b><font face="Lucida Grande" size="11">A note on efficiency:</font></b></p>
<p>We've done a couple of things in this handler to speed it up. One is to place the list returned by the <font face="Courier" size="11">openStacks</font><font face="helvetica" size="12"> function in a variable. We could have instead checked the </font><font face="Courier" size="11">openStacks</font><font face="helvetica" size="12"> directly by omitting the line put the </font><font face="Courier" size="11">openStacks</font><font face="helvetica" size="12"> into openStacksList and changing the first line of the </font><font face="Courier" size="11">repeat</font><font face="helvetica" size="12"> loop to this:</font></p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">repeat</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#980517">for</font><font face="Courier" size="11"> each line thisStack in the </font><font face="Courier" size="11" color="#F88017">openStacks</font></p>
<p></p>
<p>Done this way, the handler still works the same way. But the <font face="Courier" size="11">repeat</font><font face="helvetica" size="12"> loop must retrieve a stack name once for each open stack, and it's faster to check the contents of a variable than to check the value returned by a function.</font></p>
<p></p>
<p>The other thing we've done to make this handler faster is to use the repeat for each form of the <font face="Courier" size="11">repeat</font><font face="helvetica" size="12"> control structure. A more traditional way to write the repeat loop would be like this:</font></p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">repeat</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#980517">with</font><font face="Courier" size="11"> x = 1 to the </font><font face="Courier" size="11" color="#FF0000">number</font><font face="Courier" size="11"> of lines in openStacksList</font></p>
<p><font face="Courier" size="11">    </font><font face="Courier" size="11" color="#980517">if</font><font face="Courier" size="11"> the </font><font face="Courier" size="11" color="#FF0000">label</font><font face="Courier" size="11"> of stack (line x of openStacksList) \</font></p>
<p><font face="Courier" size="11">       is theTitle </font><font face="Courier" size="11" color="#980517">then</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#0000FF">return</font><font face="Courier" size="11"> true</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">end</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#980517">repeat</font></p>
<p></p>
<p>Many languages support this form of <font face="Courier" size="11">repeat</font><font face="helvetica" size="12"> loop, and Transcript does too: this form will work the same way as the one we used. However, the </font><font face="Courier" size="11">repeat for each</font><font face="helvetica" size="12"> form is much faster--depending on the circumstances, up to hundreds of times faster--than the repeat with x = start to end form. This handler normally has to check only a few open stacks, rather than repeating hundreds or thousands of times, so these speedups may not make much practical difference in this case. But these are good habits to learn for other handlers that are more time-consuming.</font></p>
