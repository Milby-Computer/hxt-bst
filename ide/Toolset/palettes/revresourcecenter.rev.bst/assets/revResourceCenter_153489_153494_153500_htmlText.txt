<p></p>
<p><font face="Lucida Grande" size="12" color="#000000">&bull; Background image including the border, gallows and pumpkin</font></p>
<p><font face="Lucida Grande" size="12" color="#000000">&bull; Six graphics making up the skeleton, grouped together</font></p>
<p><font face="Lucida Grande" size="12" color="#000000">&bull; Buttons to start a new game and quit</font></p>
<p><font face="Lucida Grande" size="12" color="#000000">&bull; Images to show if it is a film, book etc</font></p>

<p><font face="Lucida Grande" size="12" color="#000000">&bull; 26 buttons each representing a letter of the alphabet again grouped together.rouped together.</font></p>
<p></p>
<p><font face="Lucida Grande" size="12" color="#000000">We ensure that we name all our controls sensibly. The buttons and lines in the answer group are named for their position in the group eg button "letter11" is the button in the first column of the first line. It is always good practice to name controls well but in this case it is particularly important as we need to use the names of the controls to set up our games.</font></p>
<p></p>
<p><b><font face="Lucida Grande" size="14" color="#990006">Coding the application</font></b></p>
<p><font face="Lucida Grande" size="12" color="#000000">Now that we have our GUI in place we can start writing code to make our game playable. Earlier we decided what the game needed to do. Now we will plan this further to make the game as easy to code as possible. There are things to consider, such as are there any actions that we might want to do in more than one place? If there are it is best to write a handler to do these actions which we can then call whenever it is needed without rewriting code. We also want to consider how best to write the code to make it simple and readable.</font></p>
<p></p>
<p><font face="Lucida Grande" size="12" color="#000000">A good example of this is clearing the board. There are a number of actions we need to do to clear the board and reset the game state so we would include these in a handler which we can call whenever this needs done. Here is some pseudocode for clearing the board</font></p>
<p></p>
<p><font face="Lucida Grande" size="12" color="#000000">on resetBoard</font></p>
<p><font face="Lucida Grande" size="12" color="#000000">hide all graphics in group "skeleton"</font></p>
<p><font face="Lucida Grande" size="12" color="#000000">set all the button in group "alphabet" to show as unused</font></p>
<p><font face="Lucida Grande" size="12" color="#000000">clear the puzzle/answer area</font></p>
<p><font face="Lucida Grande" size="12" color="#000000">reset properties storing the game state</font></p>
<p><font face="Lucida Grande" size="12" color="#000000">end resetBoard</font></p>
<p></p>
<p><font face="Lucida Grande" size="12" color="#000000">Other things to consider before starting to code are</font></p>
<p></p>
<p><b><font face="Lucida Grande" size="12" color="#000000">&bull; Where do the answers come from?</font></b></p>
<p><font face="Lucida Grande" size="12" color="#000000">   I have stored a list of answers as a custom property of the stack. The application can pick a random line from this custom property to be the answer.</font></p>
<p><b><font face="Lucida Grande" size="12" color="#000000">&bull; How do we check letters?</font></b></p>
<p><font face="Lucida Grande" size="12" color="#000000">   When setting up an answer I set a custom property on each button in group "answer" to store what letter is in that position in the answer. When the user chooses an answer these custom properties are checked. If the chosen letter appears in the answer those buttons are shown, if not the next part of the skeleton is shown.</font></p>
<p><b><font face="Lucida Grande" size="12" color="#000000">&bull; How do we keep track of the game start?</font></b></p>
<p><b><font face="Lucida Grande" size="12" color="#000000">   </font></b><font face="Lucida Grande" size="12" color="#000000">We use script local variables on the card script to store how many correct and incorrect letters the user has chosen. These variables are available to all the handlers on the card script so we can check if all the letters in the answer have been found or if all the parts of the skeleton have been shown.</font></p>
<p><b><font face="Lucida Grande" size="12" color="#000000">&bull; Can we avoid duplicating code on our Letter buttons in the Alphabet Group?</font></b></p>
<p><b><font face="Lucida Grande" size="12" color="#000000">   </font></b><font face="Lucida Grande" size="12" color="#000000">Yes.  As all these buttons do the same thing there is no need to repeat code on each button. We can put the code into the script of group "Alphabet" and use the target function to identify which letter was clicked. We also need to ensure all our answers will fit within the answer area so none of the ones I have used have more than 4 words and no words have more than 7 letters. than 7 letters.</font></p>

<p><b><font face="Lucida Grande" size="14" color="#990006">Skinning the stack</font></b></p>
<p><font face="Lucida Grande" size="12" color="#000000">Once you have written all your code and tested your game and are happy with it you can add more images to make it look even better. We have used images for the letter buttons and to display the answer, for the hints and for the New Game and Quit buttons.</font></p>
<p></p>
<p><font face="Lucida Grande" size="12" color="#000000">To use images on buttons we store the images out of sight on a second card. Here we import all the images we want using the Import as Controls option in the file menu. We then set the icon of the button to the ID of the image we want to appear on it. In Revolution Media buttons can have normal icons, disabled icons, hover icons (shown when the mouse is over the button), Hilite icons (show when the mouse is down on the button) and others but these are the icons we will use.</font></p>
<p></p>
<p><font face="Lucida Grande" size="12" color="#000000">To ensure our icons display correctly we turn off a number of properties of the buttons including Show Name, Opaque, Three D, Border and Hilite Border.</font></p>
<p></p>
<p><font face="Lucida Grande" size="12" color="#000000">The background image can be imported directly onto the card as an image. Don't forget to set it to be the bottom layer so all your other controls appear on top of it.</font></p>
