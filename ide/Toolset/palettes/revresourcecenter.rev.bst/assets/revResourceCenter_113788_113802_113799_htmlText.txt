<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>This custom <font face="Courier" size="11">setProp</font><font face="helvetica" size="12"> handler is executed when you set the field's "displayedFormat" custom property, as in the following statement:</font></p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">set</font><font face="Courier" size="11"> the displayedFormat of field "Message Body" to "HTML"</font></p>
<p></p>
<p>When you set the custom property in any script or the Message Box, it triggers this <font face="Courier" size="11">setProp</font><font face="helvetica" size="12"> handler, which displays the field in the specified format--either styled text or the HTML equivalent.</font></p>
<p></p>
<p>This handler is based on the <font face="Courier" size="11">htmlText</font><font face="helvetica" size="12"> property. The </font><font face="Courier" size="11">htmlText</font><font face="helvetica" size="12"> of a field contains all of the field's text, plus style information in the form of HTML-like tags. For example, the start of boldfaced text is indicated with a &lt;b&gt; tag, and the end of the underlining is indicated with a &lt;/b&gt; tag. If a field's text looks like this:</font></p>
<p></p>
<p>  This is some bold text.</p>
<p></p>
<p>then the field's <font face="Courier" size="11">htmlText</font><font face="helvetica" size="12"> looks like this:</font></p>
<p></p>
<p>  &lt;p&gt;This is some &lt;b&gt;bold&lt;/b&gt; text.&lt;/p&gt;</p>
<p></p>
<p>The first thing the handler does is to check the value that the "displayedFormat" property is being set to. If the value is "HTML", the handler displays the field's htmlText. It does this by setting the field's text to its <font face="Courier" size="11">htmlText</font><font face="helvetica" size="12">. Because the field's </font><font face="Courier" size="11">htmlText</font><font face="helvetica" size="12"> contains all the field's contents, along with tags indicating style information, this has the effect of displaying all the field's text and styling in the field, in HTML format.</font></p>
<p></p>
<p>If the value is anything other than "HTML", the handler sets the field's <font face="Courier" size="11">htmlText</font><font face="helvetica" size="12"> to the field contents. Because the </font><font face="Courier" size="11">htmlText</font><font face="helvetica" size="12"> includes the text and styles, the field then contains styled text. If you enter some styled text into the field, then set its "displayedFormat" to "HTML" and then set it back to "styled" without changing anything, the field contains the styled text you originally entered.</font></p>
<p></p>
<p>The target function refers to the object whose custom property is being set. For example, if you trigger this handler with the statement</p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">set</font><font face="Courier" size="11"> the displayedFormat of field "Message Body" to "HTML"</font></p>
<p>then the field "Message Body" is the target. The use of the target function means you can place this handler in a stack script to apply to all the fields in that stack, or in your application's main stack script to apply to all fields throughout your application's stacks.</p>
<p></p>
<p><b><font face="Lucida Grande" size="11">Note:</font></b><font face="helvetica" size="12">  If you examine the field's custom properties after setting the "displayedFormat" property, you'll find that the custom property hasn't actually been set. This happens because the </font><font face="Courier" size="11">setProp</font><font face="helvetica" size="12"> handler traps the call to set the property; unless you pass the message, the property isn't passed to Revolution, and the property isn't set.</font></p>
<p></p>
<p>A custom property like this one, which doesn't need to actually be set in order to have an effect, is called a virtual property. Virtual properties are never attached to the object. Instead, they act to trigger <font face="Courier" size="11">setProp</font><font face="helvetica" size="12"> (or </font><font face="Courier" size="11">getProp</font><font face="helvetica" size="12">) handlers that do the actual work. In this example handler, the statement</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">set</font><font face="Courier" size="11"> the displayFormat of field "Data" to "HTML"</font></p>
<p>triggers the <font face="Courier" size="11">setProp</font><font face="helvetica" size="12"> handler, which changes the field's displayed format. Since the handler has already done everything necessary, there's no need to actually attach the custom property to the field; the statement that sets the "displayFormat" property simply acts to trigger the </font><font face="Courier" size="11">setProp</font><font face="helvetica" size="12"> handler.</font></p>
<p></p>
<p>For this example, this is fine, because we've done all we need to do in the <font face="Courier" size="11">setProp</font><font face="helvetica" size="12"> handler, and we don't need to check the property's current setting later on. So there's no need to actually set the custom property. But what if we do want to set the property, in addition to performing the actions in this handler? We can simply insert a </font><font face="Courier" size="11">pass</font><font face="helvetica" size="12"> control structure at the end of the handler:</font></p>
<p></p>
<p><font face="Courier" size="11">    ...</font></p>
<p><font face="Courier" size="11">    end if</font></p>
<p><font face="Courier" size="11">    pass displayedFormat</font></p>
<p><font face="Courier" size="11">  end displayedFormat</font></p>
<p></p>
<p>The <font face="Courier" size="11">pass</font><font face="helvetica" size="12"> control structure allows the custom property to be passed to Revolution, which sets the property. If the handler includes this line, and you set a field's "displayedFormat" property, the custom property will be changed and you'll be able to see it in the field's property inspector.</font></p>
<p></p>
<p><img src="202651" char=" "> <b><font face="Lucida Grande" size="11">Tip:</font></b><font face="helvetica" size="12"> Custom properties cannot re-use the name of a built-in property (or any other reserved word). Transcript has no "displayedFormat" property, so the word can be used as a custom property name. However, it's possible that a future version of Revolution will include a built-in property or function with the same name, and if that happens, you will need to rewrite all the handlers that use that custom property to work with the new version. To avoid any potential problem, some scripters make a point of using custom property names that are unlikely to be used by Transcript in the future. For example, some scripters prefix all their custom properties with a "c", as in "cDisplayedFormat".
