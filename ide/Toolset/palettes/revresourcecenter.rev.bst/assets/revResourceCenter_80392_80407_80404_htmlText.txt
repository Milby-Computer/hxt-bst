<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>This custom <font face="Courier" size="11">setProp</font><font face="helvetica" size="12"> handler is executed when you set an object's "redChannel" custom property, as in one of the following statements:</font></p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">set</font><font face="Courier" size="11"> the redChannel of graphic "Example" to 255</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">set</font><font face="Courier" size="11"> the redChannel of the </font><font face="Courier" size="11" color="#F88017">target</font><font face="Courier" size="11"> to myResult</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">set</font><font face="Courier" size="11"> the redChannel of </font><font face="Courier" size="11" color="#F88017">me</font><font face="Courier" size="11"> to the </font><font face="Courier" size="11" color="#FF0000">thumbPosition</font><font face="Courier" size="11"> of scrollbar 1</font></p>
<p></p>
<p>When you set the custom property in any script or the Message Box, it triggers this <font face="Courier" size="11">setProp</font><font face="helvetica" size="12"> handler, which increases or decreases the amount of red in the object's backgroundColor. For example, if you set an object's redChannel to 255, it becomes as red as possible. If you set its redChannel to zero, the object's backgroundColor has no red at all.</font></p>
<p></p>
<p>The handler works by getting the object's <font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12">, changing it to an RGB triplet if necessary, and replacing the first item in the RGB triplet--the amount of red--with the "redChannel" value.</font></p>
<p></p>
<p>The first thing the handler does is check the newValue parameter to make sure it's between zero and 255, the allowable range of values in an RGB triplet:</p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">put</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#F88017">max</font><font face="Courier" size="11">(min(newValue,255),zero) into newValue</font></p>
<p></p>
<p>It's easiest to understand this statement by taking it one part at a time, starting with the innermost part of the expression. The expression min(newValue,255) evaluates to either 255 or newValue, whichever is smaller. In other words, if newValue is too big, evaluating this part of the statement sets it to 255, and leaves it alone otherwise. Then, the expression max(min(newValue,255),zero) evaluates to either the value we just found, or zero, whichever is larger. The expression, therefore, evaluates to zero if newValue is less than zero, 255 if newValue is greater than 255, or to newValue itself if it's within the range 0&ndash;255. We put this value back into newValue, and now we're sure we're dealing with a legal value.</p>
<p></p>
<p>The demo stack doesn't need this check, because it uses a scale bar to set the channel value, so the user can't enter an illegal newValue. But it's good practice to put it into the handler anyway, because you may use this handler in other settings--for example, asking the user to type a number--where the newValue might be out of range.</p>
<p></p>
<p>Next, the handler gets the <font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> of the target object. The target function refers to the object whose custom property is being set. For example, if you trigger this handler with the statement</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">set</font><font face="Courier" size="11"> the redChannel of field "Example" to 127</font></p>
<p>then the field "Example" is the target. The use of the target function means you can place this handler in a stack script to apply to all the objects in that stack, or in your application's main stack script to apply to all objects throughout your application's stacks.</p>
<p></p>
<p>The <font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> of an object can be in any of the three equivalent forms of a color reference: a numeric RGB triplet like "255,248,198", an HTML-style color like "#FFF8C6", or a color name like "LemonChiffon". This handler is designed to work with the first kind, an RGB triplet, and if the </font><font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> is in one of the other two forms--or if the object's </font><font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> isn't set and is </font><font face="Courier" size="11">empty</font><font face="helvetica" size="12">--an execution error will result.</font></p>
<p></p>
<p>To prevent this from happening, the handler checks whether the <font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> is an RGB triplet by checking whether it has three items. If not, it uses a trick to change the </font><font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> to its RGB triplet equivalent. This trick is described in the example "Translating a color name to an RGB numeric triplet". When you set the object's </font><font face="Courier" size="11">backgroundPixel</font><font face="helvetica" size="12"> property to itself, its </font><font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> property is changed to RGB triplet format. We set the object's </font><font face="Courier" size="11">backgroundPixel</font><font face="helvetica" size="12"> to its </font><font face="Courier" size="11">effective backgroundPixel</font><font face="helvetica" size="12"> in case the </font><font face="Courier" size="11">backgroundPixel</font><font face="helvetica" size="12"> property (and therefore its </font><font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12">) is </font><font face="Courier" size="11">empty</font><font face="helvetica" size="12">. Finally, we get the </font><font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12">, which is now in RGB triplet form.</font></p>
<p></p>
<p>The first item in an RGB triplet is the intensity level of the red channel in the color. Now that we have the RGB triplet as well as the new red value, we can simply put the newValue into item 1 of the RGB triplet, and then set the object's <font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12"> to the modified triplet.</font></p>
<p></p>
<p>An obvious variation on this handler is to set the green or blue channel instead. This is done simply by changing the item number in the following line:</p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">put</font><font face="Courier" size="11"> theValue into item 1 of it</font></p>
<p></p>
<p>Item 2 is the amount of green, and item 3 is the amount of blue.</p>
<p></p>
<p>Another variation is to change a color other than the <font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12">. For example, you can change the </font><font face="Courier" size="11">foregroundColor</font><font face="helvetica" size="12"> to change the text of a field or button, using the same technique as in this example. Just substitute the </font><font face="Courier" size="11">foregroundPixel</font><font face="helvetica" size="12"> property for the </font><font face="Courier" size="11">backgroundPixel</font><font face="helvetica" size="12">, as well as substituting the </font><font face="Courier" size="11">foregroundColor</font><font face="helvetica" size="12"> for the </font><font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12">.</font></p>
<p></p>
<p>You can also include a <font face="Courier" size="11">getProp</font><font face="helvetica" size="12"> handler to allow querying the object's "redChannel" property. Such a </font><font face="Courier" size="11">getProp</font><font face="helvetica" size="12"> handler is identical to the first part of the example </font><font face="Courier" size="11">setProp</font><font face="helvetica" size="12"> handler:</font></p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">getProp</font><font face="Courier" size="11"> redChannel</font></p>
<p><font face="Courier" size="11">    </font><font face="Courier" size="11" color="#0000FF">get</font><font face="Courier" size="11"> the </font><font face="Courier" size="11" color="#FF0000">backgroundColor</font><font face="Courier" size="11"> of the </font><font face="Courier" size="11" color="#F88017">target</font></p>
<p><font face="Courier" size="11">    </font><font face="Courier" size="11" color="#980517">if</font><font face="Courier" size="11"> the </font><font face="Courier" size="11" color="#FF0000">number</font><font face="Courier" size="11"> of items of it is not 3 </font><font face="Courier" size="11" color="#980517">then</font></p>
<p><font face="Courier" size="11">      </font><font face="Courier" size="11" color="#68228B">-- use trick to convert to RGB triplet:</font></p>
<p><font face="Courier" size="11">      </font><font face="Courier" size="11" color="#0000FF">set</font><font face="Courier" size="11"> the </font><font face="Courier" size="11" color="#FF0000">backgroundPixel</font><font face="Courier" size="11"> of the </font><font face="Courier" size="11" color="#F88017">target</font><font face="Courier" size="11"> \</font></p>
<p><font face="Courier" size="11">        to the </font><font face="Courier" size="11" color="#FF0000">effective</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#FF0000">backgroundPixel</font><font face="Courier" size="11"> of the </font><font face="Courier" size="11" color="#F88017">target</font></p>
<p><font face="Courier" size="11">      </font><font face="Courier" size="11" color="#0000FF">get</font><font face="Courier" size="11"> the </font><font face="Courier" size="11" color="#FF0000">backgroundColor</font><font face="Courier" size="11"> of the </font><font face="Courier" size="11" color="#F88017">target</font></p>
<p><font face="Courier" size="11">    </font><font face="Courier" size="11" color="#980517">end</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#980517">if</font></p>
<p><font face="Courier" size="11">    </font><font face="Courier" size="11" color="#0000FF">return</font><font face="Courier" size="11"> item 1 of it </font><font face="Courier" size="11" color="#68228B">-- the red channel</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">end</font><font face="Courier" size="11"> redChannel</font></p>
<p></p>
<p><b><font face="Lucida Grande" size="11">Note:</font></b><font face="helvetica" size="12">  If you examine the object's custom properties after setting the "redChannel" property, you'll find that the custom property hasn't actually been created. This happens because the </font><font face="Courier" size="11">setProp</font><font face="helvetica" size="12"> handler traps the call to set the property; unless you pass the </font><font face="Courier" size="11">setProp</font><font face="helvetica" size="12"> trigger, the property isn't passed to Revolution, and the property isn't set.</font></p>
<p></p>
<p>A custom property like this one, which doesn't need to actually be set in order to have an effect, is called a virtual property. Virtual properties are never attached to the object. Instead, they act to trigger <font face="Courier" size="11">setProp</font><font face="helvetica" size="12"> (or </font><font face="Courier" size="11">getProp</font><font face="helvetica" size="12">) handlers that do the actual work. In this example handler, the statement</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">set</font><font face="Courier" size="11"> the redChannel of last card to storedRedNumber</font></p>
<p>triggers the <font face="Courier" size="11">setProp</font><font face="helvetica" size="12"> handler, which changes the object's </font><font face="Courier" size="11">backgroundColor</font><font face="helvetica" size="12">. Since the handler has already done everything necessary, there's no need to actually attach the custom property to the field; the statement that sets the "redChannel" property simply acts to trigger the </font><font face="Courier" size="11">setProp</font><font face="helvetica" size="12"> handler.</font></p>
