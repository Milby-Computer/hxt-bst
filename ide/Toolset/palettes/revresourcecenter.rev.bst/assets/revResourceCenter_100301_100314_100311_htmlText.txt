<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>The <font face="Courier" size="11">menuPick</font><font face="helvetica" size="12"> message is sent when the user chooses a menu item from the menu. Remember that in Revolution, menus are usually implemented as buttons:  the button's </font><font face="Courier" size="11">style</font><font face="helvetica" size="12"> and </font><font face="Courier" size="11">menuMode</font><font face="helvetica" size="12"> properties control how it is displayed, and the button's </font><font face="Courier" size="11">text</font><font face="helvetica" size="12"> property is used as the list of menu items. When the </font><font face="Courier" size="11">menuPick</font><font face="helvetica" size="12"> message is sent to the button, it triggers this handler.  For more details on how to create a menu bar, see the section on </font><i><font face="Lucida Grande" size="11">Building a User Interface</font></i><font face="helvetica" size="12"> in the </font><i><font face="Lucida Grande" size="11">User's Guide</font></i><font face="helvetica" size="12">, and for scripting menus see the section on </font><i><font face="Lucida Grande" size="11">Scripting Menus and the Menu Bar</font></i><font face="helvetica" size="12"> in the </font><i><font face="Lucida Grande" size="11">User's Guide</font></i><font face="helvetica" size="12">.</font></p>
<p></p>
<p>In the <font face="Courier" size="11">mouseDown</font><font face="helvetica" size="12"> handler described in "Updating a menu with a list of open Windows", we placed the title of each open stack in the </font><font face="Courier" size="11">menu</font><font face="helvetica" size="12">, and set a custom property called "currentStacksList" containing the corresponding stack names. If that </font><font face="Courier" size="11">mouseDown</font><font face="helvetica" size="12"> handler is in the menu button's script (or the script of the menu bar group that contains the button), the property has already been set when the user clicked the menu, so it's ready when our </font><font face="Courier" size="11">menuPick</font><font face="helvetica" size="12"> handler is executed. This property is needed because the menu displays titles, not the actual stack names; the "currentStacksList" property lets us figure out the name corresponding to the stack label the user chose.</font></p>
<p></p>
<p>The <font face="Courier" size="11">menuHistory</font><font face="helvetica" size="12"> property of a button </font><font face="Courier" size="11">menu</font><font face="helvetica" size="12"> is the number of the last-chosen menu item. Each line of the button contents becomes a separate menu item when the menu is displayed, so the </font><font face="Courier" size="11">menuHistory</font><font face="helvetica" size="12"> is also the line number of the menu item.</font></p>
<p></p>
<p>The <font face="Courier" size="11">menuPick</font><font face="helvetica" size="12"> handler also gets the text of the chosen menu item as a parameter. However, in this example, we don't really care what the text of the chosen menu item is: it's a stack </font><font face="Courier" size="11">title</font><font face="helvetica" size="12">, and you can't specify a stack by its </font><font face="Courier" size="11">title</font><font face="helvetica" size="12"> in order to bring it to the front. Instead, we need the stack </font><font face="Courier" size="11">name</font><font face="helvetica" size="12"> stored in the button's "currentStacksList" custom property.</font></p>
<p></p>
<p>The names were stored in the "currentStacksList" property in the same order as the labels in the menu itself, so if we know the menu item's number, and get that line number of the "currentStacksList" property, we have the <font face="Courier" size="11">name</font><font face="helvetica" size="12"> of the stack. The </font><font face="Courier" size="11">menuHistory</font><font face="helvetica" size="12"> gives us this number.</font></p>
<p></p>
<p>Once we've retrieved the stack's name from the "currentStacksList" property, we use the <font face="Courier" size="11">go</font><font face="helvetica" size="12"> command to bring the stack window to the front. If there is some problem and the </font><font face="Courier" size="11">go</font><font face="helvetica" size="12"> command can't find the stack, it sets the </font><font face="Courier" size="11">result</font><font face="helvetica" size="12"> function to empty. The last thing we do is check the result, and </font><font face="Courier" size="11">beep</font><font face="helvetica" size="12"> if it's not empty.</font></p>
<p></p>
<p><font color="#404040"><img src="202651" char=" "> </font><b><font face="Lucida Grande" size="11">Tip:</font></b><font face="helvetica" size="12"> The "currentStacksList" custom property is needed because the Window menu displays a list of the titles of open stacks instead of their names, for better readability, but you need the name--not the label--to refer to a stack. There's a simpler way to do this, if you don't mind displaying the name of each stack instead of its label in the Window menu. If the Window menu consists of a list of stack names, a handler to choose one of them looks like this:</font></p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">on</font><font face="Courier" size="11"> menuPick theStackName</font></p>
<p><font face="Courier" size="11">    </font><font face="Courier" size="11" color="#0000FF">go</font><font face="Courier" size="11"> stack theStackName</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">end</font><font face="Courier" size="11"> menuPick</font></p>
