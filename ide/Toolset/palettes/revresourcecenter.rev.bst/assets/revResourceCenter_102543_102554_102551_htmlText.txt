<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>This custom function is called with code such as the following:</p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">if</font><font face="Courier" size="11"> canWriteFileTo(specialFolderPath("Preferences")) </font><font face="Courier" size="11" color="#980517">then</font></p>
<p><font face="Courier" size="11">    createPrefsFile</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">else</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#0000FF">answer</font><font face="Courier" size="11"> "Could not create a Preferences file."</font></p>
<p></p>
<p>The function does not directly check the attributes of the directory. Instead, it uses a trick: it tries to create a file, using the <font face="Courier" size="11">open file</font><font face="helvetica" size="12"> command. When you use the </font><font face="Courier" size="11">open file</font><font face="helvetica" size="12"> command and specify a nonexistent file, the command creates the file.  </font><b><font face="Lucida Grande" size="11">Note:</font></b><font face="helvetica" size="12"> if you wanted to directly read the permissions attribute of a directory instead, you can use </font><b><font face="Lucida Grande" size="11">the detailed directories</font></b><font face="helvetica" size="12"> function.</font></p>
<p></p>
<p>If the command fails, indicating no file can be created in that location, <font face="Courier" size="11">the result</font><font face="helvetica" size="12"> function will return an error message. By checking whether </font><font face="Courier" size="11">the result</font><font face="helvetica" size="12"> is </font><font face="Courier" size="11">empty</font><font face="helvetica" size="12">, we can check whether the </font><font face="Courier" size="11">open file</font><font face="helvetica" size="12"> command was successful, and therefore whether it's possible to create a file at the desired location.</font></p>
<p></p>
<p>Obviously, we don't want to use a file that might already exist in the specified folder. For this reason, the handler generates a random number between 1 and 9999, using the random function. The file's name consists of the string "temp" followed by this random number. It's very unlikely that there will already be a file with such a name. Additionally, if the file is accidentally left in place after the handler finishes executing, at least its name clearly indicates that it is a temporary file and can be removed.</p>
<p></p>
<p>If <font face="Courier" size="11">the result</font><font face="helvetica" size="12"> is </font><font face="Courier" size="11">empty</font><font face="helvetica" size="12">, the command succeeded. The handler then cleans up after itself by closing and deleting the test file. If </font><font face="Courier" size="11">the result</font><font face="helvetica" size="12"> is not empty, the file wasn't created, so it doesn't need to be cleaned up.</font></p>
