<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>This custom function is called with a statement such as one of the following:</p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">get</font><font face="Courier" size="11"> reversed("A man, a canal, Panama")</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">put</font><font face="Courier" size="11"> reversed(it) into field "Reversed Text"</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">answer</font><font face="Courier" size="11"> reversed(field "Answer")</font></p>
<p></p>
<p>The "reversed" function reverses whatever string is passed to it, character by character, so that the string it returns is the "backwards" version of the original string.</p>
<p></p>
<p>The handler uses a <font face="Courier" size="11">repeat</font><font face="helvetica" size="12"> control structure to scan each character in the string. Unlike a typical repeat loop, this one doesn't start at the first character and go to the last. Instead, it uses the </font><font face="Courier" size="11">repeat</font><font face="helvetica" size="12">...</font><font face="Courier" size="11">down to</font><font face="helvetica" size="12"> form to start at the last character and go back to the first.</font></p>
<p></p>
<p>The handler places each character at the end of a variable called "reversedString". Because the repeat loop in this handler goes backwards, the "reversedString" variable ends up holding the same characters as the original string, but in reverse order. When the repeat loop finishes, the "reversedString" variable is returned as the value of the function.</p>
<p></p>
<p>You can use similar handlers to reverse the words or lines in a string. In fact, using the <font face="Courier" size="11">itemDelimiter</font><font face="helvetica" size="12"> property, you can reverse the items in a string, using any character as the item delimiter.</font></p>
<p></p>
<p><b><font face="Lucida Grande" size="11">A note on efficiency:</font></b></p>
<p>You might initially think that using the <font face="Courier" size="11">repeat for each form</font><font face="helvetica" size="12"> of the repeat control structure would be faster, since in general, this form takes much less time than the repeat with x = firstValue to lastValue form:</font></p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">repeat</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#980517">for</font><font face="Courier" size="11"> each character thisChar in theString</font></p>
<p><font face="Courier" size="11">    </font><font face="Courier" size="11" color="#0000FF">put</font><font face="Courier" size="11"> thisChar before reversedString</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">end</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#980517">repeat</font></p>
<p></p>
<p>However, this turns out to be slower in this particular case. The reason is that it takes longer to put a string before a variable than to put a string after a variable. Since we need to reverse the string, and since the <font face="Courier" size="11">repeat for each</font><font face="helvetica" size="12"> form doesn't have a way to go backwards, we have no choice but to put each character before the "reversedString" variable. The speed loss from doing this is much greater for this handler than the speed we'd gain from using the </font><font face="Courier" size="11">repeat for each</font><font face="helvetica" size="12"> form.</font></p>
