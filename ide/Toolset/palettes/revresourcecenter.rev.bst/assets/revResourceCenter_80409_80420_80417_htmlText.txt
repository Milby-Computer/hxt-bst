<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>The <font face="Courier" size="11">tabKey</font><font face="helvetica" size="12"> message is sent to the current control whenever the user presses the Tab key. For example, if the insertion point is in a field, when the user presses the Tab key, a </font><font face="Courier" size="11">tabKey</font><font face="helvetica" size="12"> message is sent to that field. When the </font><font face="Courier" size="11">tabKey</font><font face="helvetica" size="12"> message is sent to the field, it triggers this handler.</font></p>
<p></p>
<p>The handler shifts all lines that are completely or partially selected, by inserting or removing a tab at the start of each of the lines. If there is no selection but there is an insertion point, a tab character is inserted at the insertion point. If a line is already shifted all the way to the left (in other words, if it has no leading tabs), shifting it to the left doesn't affect it.</p>
<p></p>
<p>The first thing the handler does is check <font face="Courier" size="11">the selectedText</font><font face="helvetica" size="12"> function to find out whether any text is selected. If </font><font face="Courier" size="11">the selectedText</font><font face="helvetica" size="12"> is </font><font face="Courier" size="11">empty</font><font face="helvetica" size="12">, there is an insertion point but no text is selected. In this case, the handler passes the </font><font face="Courier" size="11">tabKey</font><font face="helvetica" size="12"> message on to the engine, allowing the Tab key to operate normally: as long as the field's </font><font face="Courier" size="11">tabStops</font><font face="helvetica" size="12"> property is not empty, the Tab key inserts a tab character.</font></p>
<p></p>
<p>Next, the handler finds out what part of the text is selected. The <font face="Courier" size="11">selectedLines</font><font face="helvetica" size="12"> function returns one of two forms:</font></p>
<p></p>
<p>* If more than one line is selected, it returns an expression of the form <font face="Courier" size="11">line x to y of field x</font><font face="helvetica" size="12">. In this case, the handler puts the first selected line ("x") into a variable called "firstLine", and the last selected line ("y") into a variable called "lastLine".</font></p>
<p></p>
<p>* If one line is selected, or the selection is less than one line, or there is an insertion point but nothing is selected, it returns an expression of the form <font face="Courier" size="11">line x of field z.</font><font face="helvetica" size="12"> In this case, the handler puts the selected line number into both the "firstLine" and "lastLine" variables. This simplifies the logic in the </font><font face="Courier" size="11">repeat</font><font face="helvetica" size="12"> loop later in the handler.</font></p>
<p></p>
<p>Next, the handler checks whether the user is holding down the Shift key or not. If so, the handler shifts the selected lines to the left by removing leading tabs; if not, it shifts the lines to the right by inserting tabs.</p>
<p></p>
<p>To shift left, the handler uses a <font face="Courier" size="11">repeat</font><font face="helvetica" size="12"> control structure to go through each of the lines in the selection. If the "firstLine" variable is the same as the "lastLine" variable--that is, if only one line or portion of a line is selected--the statements in the loop are executed only once. For each line, the </font><font face="Courier" size="11">repeat</font><font face="helvetica" size="12"> loop checks whether the first character is a tab. If so, it removes it. If the first character isn't a tab, that line is already shifted all the way to the left, so the handler doesn't change it.</font></p>
<p></p>
<p>Shifting right is similar, but there is no need to check whether the first character of each line is a tab: the handler simply adds a tab to the beginning of each affected line.</p>
<p></p>
<p>Finally, the handler re-selects the lines that were selected before the user pressed the Tab key. This is not strictly necessary, but it's less disorienting and more convenient for the user to retain the selection. It also helps make it clear that the Tab key affects all lines where there is even a partial selection, because the handler selects the entire line.</p>
<p></p>
<p>Many variations on this handler are possible. For example, you might prefer to shift lines using a space, or a run of several spaces, instead of a tab character. Or you might want to add or delete leading "&gt;" characters for use with email or Usenet applications, where quoted lines from previous messages are indicated using this convention.</p>
