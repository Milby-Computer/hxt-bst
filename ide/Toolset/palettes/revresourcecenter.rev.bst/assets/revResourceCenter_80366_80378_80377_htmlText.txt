<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>This script uses three handlers to control a "settable" button. Clicking the button changes the color of any text that is selected in a particular field to green. However, if you hold down the mouse over the button, a popup menu appears that lets you change the color.</p>
<p></p>
<p>The <font face="Courier" size="11">mouseDown</font><font face="helvetica" size="12"> message is sent whenever the user clicks, to whatever object was clicked. When the </font><font face="Courier" size="11">mouseDown</font><font face="helvetica" size="12"> message is sent to this button, it triggers the </font><font face="Courier" size="11">mouseDown</font><font face="helvetica" size="12"> handler. This </font><font face="Courier" size="11">mouseDown</font><font face="helvetica" size="12"> handler sends a message called "popGoesTheMenu" that pops up a menu after a </font><i><font face="Lucida Grande" size="11">two-second delay</font></i><font face="helvetica" size="12">. When the user releases the mouse, a </font><font face="Courier" size="11">mouseUp</font><font face="helvetica" size="12"> message is sent to the button, triggering the </font><font face="Courier" size="11">mouseUp</font><font face="helvetica" size="12"> handler.</font></p>
<p></p>
<p>The local declaration at the top of the script creates a variable called "didPopUpMenu". Because it is declared outside any handler, "didPopUpMenu" is a script local variable, meaning it's available to all three of the handlers in this script. This is important, since more than one handler needs to access the variable. We'll put either true or false into the "didPopUpMenu" variable, depending on whether the user triggered the popup menu.</p>
<p></p>
<p>When the user first clicks the button, a <font face="Courier" size="11">mouseDown</font><font face="helvetica" size="12"> message is sent. The </font><font face="Courier" size="11">mouseDown</font><font face="helvetica" size="12"> handler puts false into "didPopupMenu". This ensures that the variable will be set to false unless we set it to true later. In other words, this line of code sets a default value of false for the variable.</font></p>
<p></p>
<p>Next, the <font face="Courier" size="11">mouseDown</font><font face="helvetica" size="12"> handler sends the "popGoesTheMenu" message after a two-second delay. (The delay is so the user has to hold down the mouse button to see the popup menu.) This message is sent to </font><font face="Courier" size="11">me</font><font face="helvetica" size="12">. The keyword </font><font face="Courier" size="11">me</font><font face="helvetica" size="12"> refers to the current object: the object whose script contains the handler that is executing. In this example, then, the "popGoesTheMenu" message is sent to the button, because the </font><font face="Courier" size="11">mouseDown</font><font face="helvetica" size="12"> handler is in the button's script.</font></p>
<p></p>
<p>Two seconds later, the "popGoesTheMenu" handler is triggered by the delayed message. This handler displays the popup menu, so first of all, it needs to figure out whether the mouse is still being pressed and whether the mouse pointer is still inside the button. If <font face="Courier" size="11">the mouse</font><font face="helvetica" size="12"> function returns "up", the mouse button isn't down any longer. If the expression </font><font face="Courier" size="11">the mouseLoc is not within the rect of me</font><font face="helvetica" size="12"> evaluates the true, the mouse pointer isn't in the button. If either of these things is true, the popup menu shouldn't be displayed. In this case, the exit command skips the rest of the handler.</font></p>
<p></p>
<p>Otherwise, the handler uses the <font face="Courier" size="11">popup</font><font face="helvetica" size="12"> command to display the menu. (Because we're about to pop up the menu, at this point the handler puts true into the variable "didPopupMenu".) The popup menu is contained in a button called (in this example) "Colors". This is a separate button whose style property is "menu" and whose menuMode is "popup". This extra button is hidden in this example, and should be for most situations. The </font><font face="Courier" size="11">popup</font><font face="helvetica" size="12"> command displays this popup button's contents as a popup menu. When the user chooses a menu item from it, a </font><font face="Courier" size="11">menuPick</font><font face="helvetica" size="12"> message is sent to the popup button. In this example, the popup button's script looks like this:</font></p>
<p></p>
<p> <font face="Courier" size="11"> </font><font face="Courier" size="11" color="#980517">on</font><font face="Courier" size="11"> menuPick theColor</font></p>
<p><font face="Courier" size="11">    </font><font face="Courier" size="11" color="#0000FF">set</font><font face="Courier" size="11"> the </font><font face="Courier" size="11" color="#FF0000">label</font><font face="Courier" size="11"> of button "Color" to \</font></p>
<p><font face="Courier" size="11">      "Change color to" &amp;&amp; </font><font face="Courier" size="11" color="#F88017">toLower</font><font face="Courier" size="11">(theColor)</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">end</font><font face="Courier" size="11"> menuPick</font></p>
<p></p>
<p>This simply puts the color that the user picked into the main button's <font face="Courier" size="11">label</font><font face="helvetica" size="12"> property. For example, if you pick "blue" from the popup menu, the </font><font face="Courier" size="11">menuPick</font><font face="helvetica" size="12"> handler changes the main button's visible label to "Change color to blue".</font></p>
<p></p>
<p>What happens if the user just clicks, and doesn't hold down the mouse button long enough to show the menu? In this case, after the <font face="Courier" size="11">mouseDown</font><font face="helvetica" size="12"> is sent, a </font><font face="Courier" size="11">mouseUp</font><font face="helvetica" size="12"> message is sent to the button when the user releases the mouse. We don't want to perform the button action if the user popped up the menu, so we'll check that first.</font></p>
<p></p>
<p>To find out whether the popup menu was used, the <font face="Courier" size="11">mouseUp</font><font face="helvetica" size="12"> handler checks the "didPopupMenu" variable. Remember that this variable was set to false in the </font><font face="Courier" size="11">mouseDown</font><font face="helvetica" size="12"> handler, and to true in the "popGoesTheMenu" handler. If it's true, the menu was popped up. In this case, the </font><font face="Courier" size="11">pass</font><font face="helvetica" size="12"> command skips the rest of the handler and passes the </font><font face="Courier" size="11">mouseUp</font><font face="helvetica" size="12"> message on to the next object. But if it's still false, the user didn't hold down the mouse button long enough to pop up the menu, so it's time to do the button action: set the selected text's color.</font></p>
<p></p>
<p>We use <font face="Courier" size="11">pass</font><font face="helvetica" size="12"> here instead of </font><font face="Courier" size="11">exit</font><font face="helvetica" size="12"> because, since </font><font face="Courier" size="11">mouseUp</font><font face="helvetica" size="12"> is a Revolution message rather than a custom message, it's possible that there's a handler further along the message path that needs to receive this message in order to do housekeeping or other tasks. That doesn't happen to be the case for this example--there are no </font><font face="Courier" size="11">mouseUp</font><font face="helvetica" size="12"> handlers in the example card or stack--but it's a good habit to pass Revolution messages instead of exiting, unless you specifically want to prevent the message from passing.</font></p>
<p></p>
<p>Setting the text's color is fairly simple. First we check to make sure the selection is in the example field, and then we set the <font face="Courier" size="11">textColor</font><font face="helvetica" size="12"> property of the selected piece of text to the last word of the button's label. Since the popup menu changes the last word to a color name, we know that the last word of the label is always a color.</font></p>
