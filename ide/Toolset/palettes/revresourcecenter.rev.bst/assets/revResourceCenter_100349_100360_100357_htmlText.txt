<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>The <font face="Courier" size="11">menuPick</font><font face="helvetica" size="12"> message is sent when the user chooses a menu item from the menu. When the </font><font face="Courier" size="11">menuPick</font><font face="helvetica" size="12"> message is sent to the button, it triggers this handler.</font></p>
<p></p>
<p>This <font face="Courier" size="11">menuPick</font><font face="helvetica" size="12"> handler works with a menu button that's set up as a hierarchical menu, with Font, Size, and Color submenus, plus menu items for "Plain", "Bold", "Italic", and "Underline".</font></p>
<p></p>
<p>The parameter (which we've boringly named theMenuItem) that's passed with the <font face="Courier" size="11">menuPick</font><font face="helvetica" size="12"> message contains the text of the menu item the user chose. If the menu item is in a submenu, the parameter includes the submenu name and the menu item, separated by a bar (|). For example, if the user chose "Red" from the "Color" submenu, the theMenuItem parameter is "Color|Red".</font></p>
<p></p>
<p>To make it easier to separate the menu item from the submenu, the first thing the handler does is to set the itemDelimiter property to "|". By default, the <font face="Courier" size="11">itemDelimiter</font><font face="helvetica" size="12"> is a comma, but you can set it to any character. Once this </font><font face="Courier" size="11">itemDelimiter</font><font face="helvetica" size="12"> is set, if the user chose a submenu item, the submenu name is item 1 of the parameter, and the menu item is item 2 of the parameter.</font></p>
<p></p>
<p>The <font face="Courier" size="11">switch</font><font face="helvetica" size="12"> control structure that forms the heart of this handler checks the first item in the theMenuItem parameter. If the user chose a menu item from a submenu, item 1 is the name of the submenu. If the menu item is a plain menu item--not part of a submenu--then the parameter consists of the menu item's name, and "item 1" is the whole parameter. The </font><font face="Courier" size="11">switch</font><font face="helvetica" size="12"> control structure includes a clause for each top-level menu item, whether it's a submenu or a regular menu item.  For more details on the switch control structure, see the section on </font><i><font face="Lucida Grande" size="11">Making Decsions</font></i><font face="helvetica" size="12"> in the </font><i><font face="Lucida Grande" size="11">User's Guide</font></i><font face="helvetica" size="12">.</font></p>
<p></p>
<p>If the user chose a menu item from the "Font" submenu, the handler sets the field's <font face="Courier" size="11">textFont</font><font face="helvetica" size="12"> property to the font that was chosen from the submenu. (This is item 2 of the parameter.) The "Size" and "Color" submenus are handled similarly: item 2 of the parameter is the size or color the user chose from the submenu.</font></p>
<p></p>
<p>This menu has four ordinary menu items--not submenus--at the bottom: Plain, Bold, Italic, and Underline. The parameter for ordinary menu items is just the name of the menu item, and there is only one "|"-delimited item in the parameter, so the expression item 1 of theMenuItem evaluates to the name of the menu item. Because of this, the same <font face="Courier" size="11">switch</font><font face="helvetica" size="12"> control structure, which tests item 1 of theMenuItem, works for the submenus and the ordinary menu items.</font></p>
<p></p>
<p>If the user chooses "Plain", it's simple to change the style. Setting the field's <font face="Courier" size="11">textStyle</font><font face="helvetica" size="12"> property to empty removes the style information.</font></p>
<p></p>
<p><img src="202651" char=" "> <b><font face="Lucida Grande" size="11">Tip:</font></b><font face="helvetica" size="12">  You can also include an option to remove not just text styles, but all font, size, and color information as well. To do this, set the </font><font face="Courier" size="11">textFont</font><font face="helvetica" size="12"> of the field to </font><font face="Courier" size="11">empty</font><font face="helvetica" size="12"> instead of setting the </font><font face="Courier" size="11">textStyle</font><font face="helvetica" size="12">.</font></p>
<p></p>
<p>If the user chooses "Bold", "Italic", or "Underline", the handler must first figure out whether the style is already applied. If so, the style needs to be removed (without disturbing other styles); if not, it needs to be added to any existing styles. For example, if the field is already bold and underlined, and the user chooses "Underline", the underlining is removed but the boldfacing is retained.</p>
<p></p>
<p>The <font face="Courier" size="11">textStyle</font><font face="helvetica" size="12"> of an object consists of one or more styles, separated by commas. For example, the </font><font face="Courier" size="11">textStyle</font><font face="helvetica" size="12"> of a bold underlined field reports "bold,underline". To find out whether the chosen menu item is already a style of the field, the handler checks whether it's one of the items in the field's </font><font face="Courier" size="11">textStyle</font><font face="helvetica" size="12">. (Remember that earlier in the handler, we set the itemDelimiter to "|" to be able to separate submenus from their menu items. Since we've already handled all the submenus, we don't need to use this </font><font face="Courier" size="11">itemDelimiter</font><font face="helvetica" size="12"> any more, so we set it back to a </font><font face="Courier" size="11">comma</font><font face="helvetica" size="12"> in order to more easily handle the </font><font face="Courier" size="11">textStyle</font><font face="helvetica" size="12">.)</font></p>
<p></p>
<p>If the style the user chose from the menu is already a <font face="Courier" size="11">textStyle</font><font face="helvetica" size="12"> of the field, the handler removes that style. If not, it adds the style to the existing </font><font face="Courier" size="11">textStyle</font><font face="helvetica" size="12">. We need to handle one case, where no special styles are set yet, specially. If there are no styles set, the </font><font face="Courier" size="11">textStyle</font><font face="helvetica" size="12"> property reports "plain". However, if you set the </font><font face="Courier" size="11">textStyle</font><font face="helvetica" size="12"> to a string that includes "plain" as one of the items, any styles before "plain" are ignored. (For example, setting the </font><font face="Courier" size="11">textStyle</font><font face="helvetica" size="12"> to "italic,plain,bold" ignores the italic.) To avoid this problem, if the </font><font face="Courier" size="11">textStyle</font><font face="helvetica" size="12"> property reports "plain", the handler substitutes the chosen style, instead of adding it to the existing style.</font></p>
