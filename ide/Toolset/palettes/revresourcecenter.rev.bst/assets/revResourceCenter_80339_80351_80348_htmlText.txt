<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>This custom function is called with a statement such as the following:</p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">get</font><font face="Courier" size="11"> imageFileDimensions("/Disk/Folder/Picture File")</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">put</font><font face="Courier" size="11"> item 2 of imageFileDimensions(myFile) into theHeight</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">set</font><font face="Courier" size="11"> the naturalDimensions of </font><font face="Courier" size="11" color="#F88017">me</font><font face="Courier" size="11"> to imageFileDimensions(it)</font></p>
<p></p>
<p>The "imageFileDimensions" function gets the size--height and width--of a picture file in any of the formats that Revolution can import. It does this by importing the file as an image object and checking the new image's height and width properties.</p>
<p></p>
<p>When the file is imported, the new image appears on the screen. Since it's only going to be there long enough to get the height and width, we don't want the user to see the annoying flash of the image appearing and disappearing, so the first thing this handler does is to use the <font face="Courier" size="11">lock screen</font><font face="helvetica" size="12"> command to prevent changes from appearing in the window.</font></p>
<p></p>
<p>With the screen safely locked, the handler uses the <font face="Courier" size="11">import</font><font face="helvetica" size="12"> command to import the file into the stack, where it becomes a new image. The </font><font face="Courier" size="11">width</font><font face="helvetica" size="12"> and </font><font face="Courier" size="11">height</font><font face="helvetica" size="12"> of this image are </font><i><font face="Lucida Grande" size="11">automatically</font></i><font face="helvetica" size="12"> set to the picture's natural width and height, so we can check these properties to get the </font><font face="Courier" size="11">width</font><font face="helvetica" size="12"> and </font><font face="Courier" size="11">height</font><font face="helvetica" size="12"> of the original picture file. Once the handler has gotten the properties, we no longer need the imported image, so we delete it. This removes the image object we created with the import command, leaving the original picture file where it was.</font></p>
<p></p>
<p>Finally, the handler returns the width and height, separated by a comma (the , operator). The calling handler can use both, or (if it only needs one of the height or width) can check item 1 or item 2 of the return value.</p>
<p></p>
<p>What if the calling handler passed a file path that isn't in one of the formats Revolution can import--for example, what if the theFilePath is a text file? If we try to import such a file, the import command will cause an <i><font face="Lucida Grande" size="11">execution error</font></i><font face="helvetica" size="12">, and we don't want to make users deal with the Error window. (It looks sloppy.) We could check the extension of the file and reject files that don't have the proper extension, but a file might be mislabeled with the wrong extension.</font></p>
<p></p>
<p>Whenever an error occurs in a script, it sends an <font face="Courier" size="11">errorDialog</font><font face="helvetica" size="12"> message is sent to the object whose script caused the error, and the </font><font face="Courier" size="11">errorDialog</font><font face="helvetica" size="12"> handler is executed. The </font><font face="Courier" size="11">errorDialog</font><font face="helvetica" size="12"> handler in this example displays an error message to the user.</font></p>
<p></p>
<p><b><font face="Lucida Grande" size="11">Note:</font></b><font face="helvetica" size="12"> in this example, we initially set </font><font face="Courier" size="11">the lockErrorDialogs</font><font face="helvetica" size="12"> to true to prevent the Revolution debugger intercepting the error if </font><i><font face="Lucida Grande" size="11">Script Debug Mode</font></i><font face="helvetica" size="12"> is on.</font></p>
