<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>This custom function is called with a statement such as one of the following:</p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">put</font><font face="Courier" size="11"> approxEqual</font><font face="Courier" size="11" color="#000000">(firstVar,secondVar)</font><font face="Courier" size="11"> into varsAreClose</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">get</font><font face="Courier" size="11"> approxEqual(22.5,myNumber,0.1)</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">if</font><font face="Courier" size="11"> approxEqual(field 1, field 2, 1) </font><font face="Courier" size="11" color="#980517">then</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#0000FF">go</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#0000FF">next</font></p>
<p></p>
<p>Since Revolution uses binary numbers rather than decimal numbers for its internal calculations (for reasons of speed), the internal representation of a number is sometimes slightly off the correct number. This is because it's not possible to write a decimal fraction such as 0.1 exactly as a binary number: there will always be some rounding error, and this error accumulates as mathematical operations are done on the number.</p>
<p></p>
<p>For example, 10^-1 is equal to 0.1, but is calculated (to eighteen decimal places) as 0.100000000000000006. In some cases of repeated mathematical operations, this small error may accumulate to the point where it affects the final result. Setting the <font face="Courier" size="11">numberFormat</font><font face="helvetica" size="12"> property to specify many decimal places after the decimal point may also produce unexpected results in a statement that tests for an exact number. In such cases, the "approxEqual" function provides a way of testing whether numbers are close together.</font></p>
<p></p>
<p>To check whether the two numbers are approximately equal, the handler finds the difference between them using the <font face="Courier" size="11">abs</font><font face="helvetica" size="12"> function, and checks whether the difference is less than the tolerance. If so, it returns </font><font face="Courier" size="11">true</font><font face="helvetica" size="12">--the numbers are "close enough"--and if not it returns </font><font face="Courier" size="11">false</font><font face="helvetica" size="12">.</font></p>
<p></p>
<p><font color="#404040"><img src="202651" char=" "> </font><b><font face="Lucida Grande" size="11">Tip:</font></b><font face="helvetica" size="12"> The last three lines of the function--</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">if</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#F88017">abs</font><font face="Courier" size="11">(firstNumber - secondNumber) &lt; tolerance</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">then</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#0000FF">return</font><font face="Courier" size="11"> true  </font><font face="Courier" size="11" color="#68228B">-- the numbers are within tolerance</font></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#980517">else</font><font face="Courier" size="11"> </font><font face="Courier" size="11" color="#0000FF">return</font><font face="Courier" size="11"> false </font><font face="Courier" size="11" color="#68228B">-- not close enough</font></p>
<p></p>
<p>can instead be written in this shorter (though possibly harder-to-understand) form:</p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">return</font><font face="Courier" size="11"> (</font><font face="Courier" size="11" color="#000000">abs(firstNumber</font><font face="Courier" size="11"> - secondNumber</font><font face="Courier" size="11" color="#000000">)</font><font face="Courier" size="11"> &lt; tolerance</font><font face="Courier" size="11" color="#000000">) </font></p>
<p>This form does the same thing--returns true if the expression is <font face="Courier" size="11">true</font><font face="helvetica" size="12">, and </font><font face="Courier" size="11">false</font><font face="helvetica" size="12"> if it is false--but using one fewer statement.</font></p>
<p></p>
<p>The function takes three parameters--the two numbers being compared, and the amount they're allowed to differ before being considered unequal. You'll notice that you can call the function with just two parameters (the two numbers). It's possible to do this because the first line of the function handler checks to see whether there's a third parameter. If not, the handler assigns the value of 1 to the tolerance. In other words, if you don't specify a tolerance when you use the function, it assumes you want to check whether the numbers are within 1 of each other.</p>
<p></p>
<p>You can use this method with any handler to assign a default value to one or more of the parameters. Just check whether the parameter is empty. If it is, then no value has been passed, and you set the default value by simply putting the desired default into the parameter.</p>
<p></p>
<p><font color="#404040"><img src="202651" char=" "> </font><b><font face="Lucida Grande" size="11">Tip:</font></b><font face="helvetica" size="12"> This example does not check for errors in the parameters. If the function is called with incorrect parameters, an execution error may result. (For example, if any of the parameters are non-numbers, the handler will stop at the if clause with an error.) If the handler is called in a controlled way--for example, if it's called only from a handler that always passes numerical parameters--this is acceptable, but for production code, it is better to check the parameters at the top of the handler.</font></p>
