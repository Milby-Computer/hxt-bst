<p><b><font face="helvetica" size="12">Discussion:</font></b></p>
<p>This script uses three handlers to produce a slideshow that displays a series of cards. The <font face="Courier" size="11">openCard</font><font face="helvetica" size="12"> message is sent whenever a card is displayed (including when the stack first opens and one of its cards is displayed). It's sent to the card that is opening. If that card's script doesn't have an </font><font face="Courier" size="11">openCard</font><font face="helvetica" size="12"> handler, the message is passed on to the stack that the card is part of. Because the handlers in this example will be used by all the cards in the stack, the best place to put them is in the stack script.</font></p>
<p></p>
<p>The <font face="Courier" size="11">openCard</font><font face="helvetica" size="12"> handler sends a message called "showNextSlide" that goes to the next card after a two-second delay. Going to the next card, of course, sends another </font><font face="Courier" size="11">openCard</font><font face="helvetica" size="12"> message, so the handler is executed again and proceeds to the next card. The cycle continues from card to card. When the user clicks, a </font><font face="Courier" size="11">mouseUp</font><font face="helvetica" size="12"> message is sent. If the slideshow is already going, the </font><font face="Courier" size="11">mouseUp</font><font face="helvetica" size="12"> handler cancels the move to the next card. Otherwise, it starts up the slideshow again, so the user can click to pause and then click again to continue.</font></p>
<p></p>
<p>The local declaration at the top of the script creates a variable called "theMessageID". Because it is declared outside any handler, "theMessageID" is a script local variable, meaning it's available to all three of the handlers in this script. This is important, since more than one handler needs to access the variable. We'll see in a moment how this variable is used to store the information needed to pause the slideshow.</p>
<p></p>
<p>When the <font face="Courier" size="11">openCard</font><font face="helvetica" size="12"> handler is executed, it sends the "showNextSlide" message after a two-second delay. The delay is so the user has time to see the card before the next card appears. You can change the delay time, depending on the needs of your slideshow. This message is sent to </font><font face="Courier" size="11">me</font><font face="helvetica" size="12">. The keyword </font><font face="Courier" size="11">me</font><font face="helvetica" size="12"> refers to the current object: the object whose script contains the handler that is executing. In this example, then, the "showNextSlide" message is sent to the stack, because the </font><font face="Courier" size="11">openCard</font><font face="helvetica" size="12"> handler is in the stack script.</font></p>
<p></p>
<p>The <i><font face="Lucida Grande" size="11">send...in time</font></i><font face="helvetica" size="12"> form of the send command that's used here puts information about the message into a queue called the </font><font face="Courier" size="11">pendingMessages</font><font face="helvetica" size="12"> and assigns a unique number to the pending message. This number is returned in the result function, so our handler can retrieve it. A message waiting in the queue can be canceled later with the </font><font face="Courier" size="11">cancel</font><font face="helvetica" size="12"> command. Since we want to cancel the pending message if the user clicks the mouse, we need to store the message's number for later use. The next line of the handler stores the message's number in the script local variable "theMessageID", which we declared at the top of the script. For more details on timer based messaging, see the section on </font><i><font face="Lucida Grande" size="11">Timer Based Messaging</font></i><font face="helvetica" size="12"> in the User's Guide.</font></p>
<p></p>
<p>Every time Revolution opens a card, the <font face="Courier" size="11">openCard</font><font face="helvetica" size="12"> handler places a "showNextSlide" message in the </font><font face="Courier" size="11">pendingMessages</font><font face="helvetica" size="12"> queue. Two seconds later, that pending message executes, instructing Revolution to go to the next card. This triggers a new </font><font face="Courier" size="11">openCard</font><font face="helvetica" size="12"> message, which queues up a new "showNextSlide" message in the </font><font face="Courier" size="11">pendingMessages</font><font face="helvetica" size="12"> and the process starts over.</font></p>
<p></p>
<p><b><font face="Lucida Grande" size="11">Note:</font></b><font face="helvetica" size="12"> After the pending message is sent, it is removed from the </font><font face="Courier" size="11">pendingMessages</font><font face="helvetica" size="12"> queue.</font></p>
<p></p>
<p>The <font face="Courier" size="11">mouseUp</font><font face="helvetica" size="12"> handler checks the </font><font face="Courier" size="11">pendingMessages</font><font face="helvetica" size="12"> to see if the word "showNextSlide" is in the queue. If it is, the slide show is currently in action and should be stopped. All we need to do to stop the slide show is to cancel whatever pending "showNextSlide" message is waiting to be sent. Once that is canceled, the next card is not shown and the whole series of messages stops. To cancel a pending message, the mouseUp handler uses the cancel command:</font></p>
<p></p>
<p><font face="Courier" size="11">  </font><font face="Courier" size="11" color="#0000FF">cancel</font><font face="Courier" size="11"> theMessageID</font></p>
<p></p>
<p>Here is where we use the script local variable "theMessageID". This variable was declared in the first line, and in the <font face="Courier" size="11">openCard</font><font face="helvetica" size="12"> handler, we stored the message number of the pending message in the variable. The </font><font face="Courier" size="11">cancel</font><font face="helvetica" size="12"> command needs this number in order to figure out which pending message to cancel.</font></p>
<p></p>
<p>If there is no pending "showNextSlide" message, the <font face="Courier" size="11">mouseUp</font><font face="helvetica" size="12"> handler issues a "showNextSlide" message to start the slide show going again.</font></p>
